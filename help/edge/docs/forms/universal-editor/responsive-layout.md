# 범용 편집기를 사용하여 응답형 Forms 만들기

사용자는 데스크탑, 태블릿 및 스마트폰 등 다양한 디바이스의 양식에 액세스합니다. 반응형 양식을 디자인하면 디바이스에 상관없이 모든 사용자에게 최적의 경험을 제공할 수 있습니다. 이 안내서에서는 범용 편집기를 사용하여 화면 크기에 따라 양식을 디자인, 테스트 및 최적화하는 방법을 설명합니다.

반응형 양식 작성에는 두 가지 주요 활동이 포함됩니다.

- **응답형 테스트:** 장치 에뮬레이터를 사용하여 다양한 화면 크기에서 양식을 미리 보고 테스트합니다.
- **응답형 디자인:** 다양한 장치에 원활하게 적용되는 레이아웃 패턴을 선택하고 구현합니다.

**이 안내서에서 다음 방법을 배웁니다.**

- 데스크탑, 태블릿 및 모바일 장치에서 양식 테스트
- 콘텐츠에 적합한 레이아웃 패턴 선택
- 반응형 디자인 모범 사례 적용
- 일반적인 반응형 양식 문제 해결
- 모바일 성능을 위한 양식 최적화

## 반응형 Forms이 중요한 이유

**사용자 환경 영향:**

- 사용자의 60% 이상이 모바일 장치에서 양식에 액세스
- 모바일 경험이 부족하면 포기 비율이 67% 더 높아집니다
- 반응형 양식은 완료율을 최대 25%까지 높일 수 있습니다.

**비즈니스 이점:**

- 높은 양식 완료율
- 사용자 만족도 향상
- 액세스 가능성 규정 준수 향상
- 개발 및 유지 관리 비용 절감

>[!TIP]
>
> **모바일 우선 접근 방식:** 모바일 장치에 대한 디자인을 시작한 다음 더 큰 화면을 위해 기능을 향상시킵니다. 이렇게 하면 가장 제한된 환경에서 핵심 기능이 작동합니다.

## 1부: 장치 간 Forms 테스트

다른 디바이스에서 양식을 테스트하면 사용자가 응답형 문제를 접하기 전에 이를 식별하고 해결하는 데 도움이 됩니다. 범용 편집기는 다양한 화면 크기와 방향을 시뮬레이션하는 에뮬레이터 모드를 제공합니다.

### Forms을 테스트하는 방법

**1단계: 장치 에뮬레이터 열기**

1. 유니버설 편집기에서 양식을 엽니다.
2. 도구 모음에서 ![에뮬레이터 아이콘](/help/edge/docs/forms/universal-editor/assets/emulator.png){height=2%,width=2%} **에뮬레이터** 아이콘을 클릭합니다.
3. 장치 선택기 메뉴가 나타납니다.

![유니버설 편집기 반응형 테스트 인터페이스](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**2단계: 테스트할 장치 선택**

- **데스크톱**(1200px+ 너비): 기본 편집 보기
- **태블릿**(768px-1199px 너비): Medium 화면 테스트
- **모바일**(320px-767px 너비): 작은 화면 테스트
- **사용자 지정**: 특정 장치에 정확한 차원을 지정하십시오.

**3단계: 장치 방향 테스트**

**화면 회전기** 아이콘을 사용하여 다음 두 가지를 모두 테스트하십시오.

- **세로 모드**: 표준 모바일 방향
- **가로 모드**: 태블릿 또는 휴대폰 보기가 회전됨

### 장치 테스트 결과

각 디바이스 유형은 고유한 반응형 동작을 표시합니다.

| **디바이스 유형** | **화면 너비** | **확인할 내용** | **일반적인 문제** |
|-----------------|------------------|-------------------|-------------------|
| **데스크톱** | 1200px+ | 전체 레이아웃, 모든 기능 표시 | 과도한 공백, 지나치게 넓은 레이아웃 |
| **태블릿** | 768px-1199px | 구성 요소 스택, 탐색 편의성 | 크기 조정, 터치 타겟 문제 |
| **모바일** | 320px-767px | 단일 열 레이아웃, 썸네일 탐색 | 작은 텍스트, 촘촘한 간격 단추 |
| **사용자 정의** | 사용자 정의 | 장치별 요구 사항 | 중단점 극단적 사례 |

### 테스트 워크플로

**새 Forms:**

1. **데스크톱 보기에서 빌드:** 전체 기능으로 시작합니다.
2. **태블릿에서 테스트:** 중간 화면에 대한 적응을 확인합니다.
3. **모바일에서 유효성 검사:** 작은 화면에서 유용성을 확인합니다.
4. **응답형 문제 해결:** 필요에 따라 레이아웃을 조정합니다.
5. **모든 장치를 다시 테스트합니다.** 모든 크기의 수정 내용을 확인합니다.

기존 Forms의 경우 **:**

1. **빠른 모바일 확인:** 양식이 휴대폰에서 작동합니까?
2. **문제 영역 식별:** 메모 레이아웃 및 유용성 문제.
3. **체계적인 테스트:** 각 장치 크기를 철저하게 테스트합니다.
4. **문서 문제:** 수정해야 할 사항을 추적합니다.
5. **수정 사항 구현:** 규칙적으로 문제를 해결합니다.

## 2부: 응답형 레이아웃 패턴 선택

레이아웃 패턴은 양식 콘텐츠가 다양한 화면 크기에 적응하는 방법을 결정합니다. 올바른 패턴은 사용자 경험과 모바일 성능을 모두 향상시킵니다.

### 레이아웃 패턴 개요

| **레이아웃 유형** | **적합한 대상** | **모바일 성능** | **복잡성** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **패널 레이아웃** | 분류된 컨텐츠, 대시보드 스타일 양식 | 양호 | 낮음 |
| **마법사 레이아웃** | 여러 단계 프로세스, 복잡한 워크플로 | 훌륭함 | 중간 |
| **아코디언 레이아웃** | FAQ 스타일 콘텐츠, 선택 사항 섹션 | 훌륭함 | 낮음 |

### 빠른 결정 안내서

**다음 경우에 패널 레이아웃 사용:**

- 콘텐츠는 개별 카테고리로 나뉩니다
- 사용자는 여러 섹션을 한 번에 봐야 합니다
- 콘텐츠는 비교적 간단합니다

**다음 경우에 마법사 레이아웃 사용:**

- 양식에는 여러 논리적 단계가 있습니다
- 당신은 인지부하를 줄이기를 원합니다
- 모바일 사용자는 기본 대상입니다

**다음 경우에 아코디언 레이아웃 사용:**

- 양식에 선택적 또는 보조 콘텐츠가 포함되어 있습니다
- 공간 보존이 중요하다
- 컨텐츠를 논리적으로 그룹화할 수 있습니다.

### 패널 레이아웃

**목적:** 관련 콘텐츠를 동시에 볼 수 있는 시각적으로 구별되는 섹션으로 구성합니다.

![패널 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**응답형 동작:**

- **데스크톱(1200px+):** 패널은 나란히 또는 격자에 표시됩니다.
- **태블릿(768px-1199px):**&#x200B;개의 패널이 간격을 두고 세로로 스택
- **모바일(320px-767px):** 섹션 구분이 명확한 단일 열 레이아웃

**구현 단계:**

1. [패널 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel)를 사용하십시오.
2. 각 패널 내의 관련 필드를 그룹화합니다.
3. 각 섹션의 명확한 머리글을 추가합니다.
4. 패널 사이의 적절한 간격을 유지하십시오.

**모범 사례:**

- 데스크탑에서 3~4개의 패널로 제한하여 압도적인 사용자를 방지하십시오.
- 각 패널에 수사적 제목을 사용합니다.
- 인지부하를 줄이기 위해 관련 분야를 논리적으로 그룹화한다.
- 터치 장치에서 패널 탐색을 테스트합니다.

**사용 사례 예:**

- **작업 응용 프로그램:** 개인 정보, 교육, 경험, 참조
- **제품 등록:** 기본 세부 정보, 기술 사양, 무상수리 정보
- **Survey Forms:** 인구 통계, 환경 설정, 피드백, 연락처

### 마법사 레이아웃

**목적:**&#x200B;은(는) 단계별로 복잡한 프로세스를 안내하여 인지 부하를 줄이고 완료율을 향상시킵니다.

![마법사 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**응답형 동작:**

- **모든 장치:** 최적의 모바일 환경을 위해 한 단계 포커스를 유지합니다.
- **단계 콘텐츠:** 각 단계 내에서 조정됩니다(스택 또는 나란히).
- **탐색:** 간격이 충분한 터치식 단추.
- **진행률 표시기:** 화면 크기에 맞게 적절히 조절됩니다.

**구현 단계:**

1. [마법사 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard)를 사용합니다.
2. 복잡한 양식을 논리적 단계로 나눕니다(3~7단계가 최적).
3. 사용자 방향에 대한 진행 표시기를 포함합니다.
4. 명확한 탐색 컨트롤을 제공합니다(다음, 뒤로, 저장).

**모바일 최적화:**

- 탐색 단추에는 큰 터치 대상(최소 44px)을 사용합니다.
- 단계 표시기가 명확하고 작은 화면에 표시되는지 확인합니다.
- 스크롤을 줄이려면 단계당 필드 수를 제한하십시오.
- 데이터 손실을 방지하려면 자동 저장 을 활성화하십시오.

**모범 사례:**

- 논리적 단계 진행 확인 - 각 단계는 이전 단계를 기반으로 빌드되어야 합니다.
- 사용자가 예상할 내용을 알 수 있도록 명확한 단계 제목을 사용합니다.
- 오류를 조기에 발견하기 위해 각 단계에서 입력을 확인합니다.
- 사용자가 뒤로 이동하여 정보를 검토하거나 편집할 수 있습니다.

**사용 사례 예:**

- **보험 청구:**&#x200B;사고 → 증빙 → 개인 → 검토
- **계정 설정:** 기본 정보 → 환경 설정 → 보안 → 확인
- **주문 프로세스:** 제품 → 배송 → 결제 → 요약

### 아코디언 레이아웃

**목적:** 컨텐츠를 축소 가능한 섹션으로 구성하여 공간을 절약합니다. 선택적 또는 보조 정보에 이상적입니다.

![아코디언 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**응답형 동작:**

- **뛰어난 모바일 성능:** 관련 콘텐츠만 표시됩니다.
- **터치에 적합한 머리글:** 섹션을 쉽게 탭하고 확장합니다.
- **부드러운 애니메이션:** 상호 작용에 대한 시각적 피드백을 제공합니다.
- **공간 효율성:** 모든 장치에서 스크롤을 최소화합니다.

**구현 단계:**

1. [아코디언 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion)를 사용하십시오.
2. 각 섹션에서 관련 선택적 콘텐츠를 그룹화합니다.
3. 설명 섹션 머리글을 사용합니다.
4. 적절한 기본 열기/닫기 상태를 설정합니다.

**모바일 이점:**

- 사용하지 않는 섹션을 축소하여 스크롤을 줄입니다.
- 자연스러운 확장/축소 제스처와의 터치 친화적인 상호 작용.
- 더 빠른 로드 — 활성 컨텐츠만 표시됩니다.
- 집중도 향상—사용자가 필요한 것만 볼 수 있습니다.

**모범 사례:**

- 확장하기 전에 내부에 있는 내용을 사용자가 알 수 있도록 섹션 머리글을 지웁니다.
- 각 섹션 내에서 논리적으로 관련 콘텐츠를 그룹화합니다.
- 필요한 경우 확장을 시작하도록 중요 섹션을 설정합니다.
- 사용자가 확장할 항목을 결정하는 데 도움이 되는 간단한 섹션 미리보기를 제공합니다.

**사용 사례 예:**

- **제품 구성:** 기본 → 고급 → 액세서리 → 지원
- **FAQ Forms:** 계정 → 청구 → 기술 → 일반

## 포괄적인 문제 해결

### 레이아웃 문제

+++모바일에서 양식 레이아웃 나누기

**일반적인 원인:**

- 크기가 조정되지 않는 고정 너비 요소
- 데스크탑 최초 레이아웃용으로 설계된 CSS
- 해당 컨테이너를 오버플로한 이미지 또는 콘텐츠

**솔루션:**

- 이미지 및 컨테이너의 크기가 화면 크기에 맞게 조절되도록 합니다.
- 점진적 개선이 포함된 모바일 우선 디자인 을 사용하십시오.
- 장치 에뮬레이터와 실제 장치를 모두 사용하여 테스트합니다.
- 고정된 크기 대신 유연한 크기 조정을 사용하십시오.

+++

+++터치 대상이 너무 작음

**일반적인 원인:**

- 44px × 44px보다 작은 단추
- 너무 가까이 배치된 대화형 요소
- 사용자 지정 CSS가 터치 친화적 기본값 재정의

**솔루션:**

- 모든 대화형 요소가 최소 44px×44px인지 확인합니다.
- 버튼과 링크 사이에 간격을 추가합니다.
- 마우스뿐만 아니라 실제 손가락과의 터치 상호 작용을 테스트합니다.
- 터치 대상 영역을 늘려 보다 쉽게 탭할 수 있습니다.

+++

+++컨텐츠 오버플로 문제

**일반적인 원인:**

- 줄 바꿈하지 않는 긴 텍스트 또는 레이블
- 고정 폭 컨테이너
- 이미지 크기가 제대로 조정되지 않음

**솔루션:**

- 긴 콘텐츠에 대해 텍스트 줄바꿈을 활성화합니다.
- 적절히 크기가 조절되는 응답형 이미지를 사용하십시오.
- 콘텐츠에 맞게 조정할 수 있는 유연한 레이아웃을 구현합니다.
- 다양한 콘텐츠 길이로 테스트합니다.

+++

### 성능 문제

+++모바일 로드 속도 느림

**일반적인 원인:**

- 기술을 사용하여 모바일 로드 속도를 높입니다.
- 서드파티 스크립트 및 위젯을 최소화합니다.

+++
