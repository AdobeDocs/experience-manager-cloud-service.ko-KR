# 범용 편집기를 사용하여 응답형 Forms 만들기

최신 웹 환경은 끊임없이 확장되는 디바이스와 화면 크기에서 원활하게 작동하는 양식을 요구합니다. 대형 데스크탑 모니터부터 소형 스마트폰 화면에 이르기까지 사용자는 선택한 기기에 상관없이 일관되고 직관적인 경험을 기대합니다. 반응형 양식 만들기는 더 이상 선택 사항이 아니며, 전문적이고 액세스 가능하며 전환에 최적화된 디지털 경험을 제공하기 위한 기본 요구 사항입니다.

Universal Editor는 다양한 화면 차원, 입력 방법 및 사용자 컨텍스트에 맞게 지능적으로 조정되는 반응형 양식을 개발하기 위한 포괄적인 도구와 방법을 제공합니다. 이 안내서에서는 유용성, 접근성 및 시각적 호소력을 유지하면서 모든 디바이스에서 예외적으로 수행되는 양식을 만드는 데 필요한 기술 기반, 구현 전략 및 최적화 기법을 살펴봅니다.

반응형 양식 작성에는 두 가지 주요 활동이 포함됩니다.

- **응답형 테스트:** 장치 에뮬레이터를 사용하여 다양한 화면 크기에서 양식을 미리 보고 테스트합니다.
- **응답형 디자인:** 다양한 장치에 원활하게 적용되는 레이아웃 패턴을 선택하고 구현합니다.

**이 안내서에서 다음 방법을 배웁니다.**

- 데스크탑, 태블릿 및 모바일 장치에서 양식 테스트
- 콘텐츠에 적합한 레이아웃 패턴 선택
- 반응형 디자인 모범 사례 적용
- 일반적인 반응형 양식 문제 해결
- 모바일 성능을 위한 양식 최적화

## 반응형 Forms이 중요한 이유

**사용자 환경 영향:**

- 사용자의 60% 이상이 모바일 장치에서 양식에 액세스
- 모바일 경험이 부족하면 포기 비율이 67% 더 높아집니다
- 반응형 양식은 완료율을 최대 25%까지 높일 수 있습니다.

**비즈니스 이점:**

- 높은 양식 완료율
- 사용자 만족도 향상
- 액세스 가능성 규정 준수 향상
- 개발 및 유지 관리 비용 절감

>[!TIP]
>
> **모바일 우선 접근 방식:** 모바일 장치에 대한 디자인을 시작한 다음 더 큰 화면을 위해 기능을 향상시킵니다. 이렇게 하면 가장 제한된 환경에서 핵심 기능이 작동합니다.

## 1부: 장치 간 Forms 테스트

다른 디바이스에서 양식을 테스트하면 사용자가 응답형 문제를 접하기 전에 이를 식별하고 해결하는 데 도움이 됩니다. 범용 편집기는 다양한 화면 크기와 방향을 시뮬레이션하는 에뮬레이터 모드를 제공합니다.

### Forms을 테스트하는 방법

**1단계: 장치 에뮬레이터 열기**

1. 유니버설 편집기에서 양식을 엽니다.
2. 도구 모음에서 ![에뮬레이터 아이콘](/help/edge/docs/forms/universal-editor/assets/emulator.png){height=2%,width=2%} **에뮬레이터** 아이콘을 클릭합니다.
3. 장치 선택기 메뉴가 나타납니다.

![유니버설 편집기 반응형 테스트 인터페이스](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**2단계: 테스트할 장치 선택**

- **데스크톱**(1200px+ 너비): 기본 편집 보기
- **태블릿**(768px-1199px 너비): Medium 화면 테스트
- **모바일**(320px-767px 너비): 작은 화면 테스트
- **사용자 지정**: 특정 장치에 정확한 차원을 지정하십시오.

**3단계: 장치 방향 테스트**

**화면 회전기** 아이콘을 사용하여 다음 두 가지를 모두 테스트하십시오.

- **세로 모드**: 표준 모바일 방향
- **가로 모드**: 태블릿 또는 휴대폰 보기가 회전됨

### 장치 테스트 결과

각 디바이스 유형은 고유한 반응형 동작을 표시합니다.

| **디바이스 유형** | **화면 너비** | **확인할 내용** | **일반적인 문제** |
|-----------------|------------------|-------------------|-------------------|
| **데스크톱** | 1200px+ | 전체 레이아웃, 모든 기능 표시 | 과도한 공백, 지나치게 넓은 레이아웃 |
| **태블릿** | 768px-1199px | 구성 요소 스택, 탐색 편의성 | 크기 조정, 터치 타겟 문제 |
| **모바일** | 320px-767px | 단일 열 레이아웃, 썸네일 탐색 | 작은 텍스트, 촘촘한 간격 단추 |
| **사용자 정의** | 사용자 정의 | 장치별 요구 사항 | 중단점 극단적 사례 |

### 테스트 워크플로

**새 Forms:**

1. **데스크톱 보기에서 빌드:** 전체 기능으로 시작합니다.
2. **태블릿에서 테스트:** 중간 화면에 대한 적응을 확인합니다.
3. **모바일에서 유효성 검사:** 작은 화면에서 유용성을 확인합니다.
4. **응답형 문제 해결:** 필요에 따라 레이아웃을 조정합니다.
5. **모든 장치를 다시 테스트합니다.** 모든 크기의 수정 내용을 확인합니다.

기존 Forms의 경우 **:**

1. **빠른 모바일 확인:** 양식이 휴대폰에서 작동합니까?
2. **문제 영역 식별:** 메모 레이아웃 및 유용성 문제.
3. **체계적인 테스트:** 각 장치 크기를 철저하게 테스트합니다.
4. **문서 문제:** 수정해야 할 사항을 추적합니다.
5. **수정 사항 구현:** 규칙적으로 문제를 해결합니다.

## 2부: 응답형 레이아웃 패턴 선택

레이아웃 패턴은 양식 콘텐츠가 다양한 화면 크기에 적응하는 방법을 결정합니다. 올바른 패턴은 사용자 경험과 모바일 성능을 모두 향상시킵니다.

### 레이아웃 패턴 개요

| **레이아웃 유형** | **적합한 대상** | **모바일 성능** | **복잡성** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **패널 레이아웃** | 분류된 컨텐츠, 대시보드 스타일 양식 | 양호 | 낮음 |
| **마법사 레이아웃** | 여러 단계 프로세스, 복잡한 워크플로 | 훌륭함 | 중간 |
| **아코디언 레이아웃** | FAQ 스타일 콘텐츠, 선택 사항 섹션 | 훌륭함 | 낮음 |

### 빠른 결정 안내서

**다음 경우에 패널 레이아웃 사용:**

- 콘텐츠는 개별 카테고리로 나뉩니다
- 사용자는 여러 섹션을 한 번에 봐야 합니다
- 콘텐츠는 비교적 간단합니다

**다음 경우에 마법사 레이아웃 사용:**

- 양식에는 여러 논리적 단계가 있습니다
- 당신은 인지부하를 줄이기를 원합니다
- 모바일 사용자는 기본 대상입니다

**다음 경우에 아코디언 레이아웃 사용:**

- 양식에 선택적 또는 보조 콘텐츠가 포함되어 있습니다
- 공간 보존이 중요하다
- 컨텐츠를 논리적으로 그룹화할 수 있습니다.

### 패널 레이아웃

패널 레이아웃은 관련 컨텐츠를 시각적으로 구별되는 섹션으로 구성하여 사용자가 여러 섹션을 한 번에 볼 수 있도록 합니다. 이 레이아웃은 큰 화면에서 나란히 표시되는 프레젠테이션의 이점이 있는 분류된 정보가 있는 양식에 이상적입니다.

![패널 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**응답형 동작**

- **데스크톱(1200px 이상):** 패널은 최대 가시성을 위해 나란히 또는 격자로 표시됩니다.
- **태블릿(768px-1199px):** 패널이 적절한 간격으로 수직으로 스택되어 명확성을 유지합니다.
- **모바일(320px-767px):** 패널은 쉽게 탐색할 수 있도록 섹션 간의 명확한 구분이 있는 단일 열 레이아웃으로 표시됩니다.

**구현 방법**

1. 양식에 [패널 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel)를 추가합니다.
2. 각 패널 내의 관련 필드를 그룹화하여 논리 구성을 유지 관리합니다.
3. 각 패널 섹션에 명확하고 설명적인 제목을 지정합니다.
4. 패널 사이의 간격이 충분하여 화면이 흐려지지 않는지 확인하십시오.

**모범 사례**

- 데스크탑에서 패널 수를 3 또는 4로 제한하여 사용자가 압도적이지 않도록 합니다.
- 각 패널에 대해 간결하고 설명적인 제목을 사용하여 사용자의 이해를 돕습니다.
- 패널 내의 필드를 논리적으로 구성하여 인지 부하를 최소화합니다.
- 터치 장치에서 패널 탐색을 테스트하여 모든 플랫폼에서 유용성을 보장합니다.

**일반적인 사용 사례**

- **Job 지원:** 개인 정보, 교육, 경험 및 참조에 대한 섹션.
- **제품 등록:** 패널(기본 세부 정보, 기술 사양 및 보증 정보).
- 인구 통계, 환경 설정, 피드백 및 연락처 정보를 위한 **Survey Forms:** 그룹화.

### 마법사 레이아웃

마법사 레이아웃은 한 번에 한 섹션씩 표시되는 여러 단계 프로세스를 안내합니다. 이 레이아웃은 프로세스를 관리 가능한 단계로 분할하여 인지 부하를 줄이고 완료율을 높이므로 복잡한 양식에 특히 효과적입니다.

![마법사 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**응답형 동작**

- **모든 장치:** 모바일 사용자에게 최적화된 한 단계 포커스를 유지합니다.
- **단계 콘텐츠:** 각 단계는 화면 크기에 맞게 필드를 겹치거나 나란히 배열하여 응답형으로 조정됩니다.
- **탐색:** 손쉬운 상호 작용을 위해 적절한 간격이 있는 터치 전용 단추를 제공합니다.
- **진행률 표시기:** 진행률 표시줄 또는 단계 표시기는 다른 장치에 맞게 적절히 조정되어 완료 상태에 대한 명확한 피드백을 제공합니다.

**구현 방법**

1. [마법사 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard)를 양식에 삽입합니다.
2. 양식을 논리 단계, 이상적으로 3과 7 사이로 나누어 각 단계에 초점을 맞추고 관리할 수 있도록 합니다.
3. 진행 표시기를 추가하여 사용자가 프로세스에서 자신의 위치를 이해하는 데 도움이 됩니다.
4. 다음, 뒤로 및 저장 단추와 같은 명확한 탐색 컨트롤을 제공합니다.

**모바일 최적화 팁**

- 탐색 컨트롤에 큰 터치 대상(높이 44px 이상)을 사용하여 접근성을 높입니다.
- 작은 화면에 단계 표시기가 표시되고 읽을 수 있는지 확인하십시오.
- 스크롤을 최소화하고 포커스를 향상시키려면 단계당 필드 수를 제한하십시오.
- 사용자가 양식을 떠날 때 데이터 손실을 방지하기 위해 자동 저장 기능을 활성화합니다.

**모범 사례**

- 각 단계가 이전 단계를 기반으로 구축되면서 논리적 진행률을 따르는 설계 단계를 수행합니다.
- 각 단계에 대해 명확하고 설명적인 제목을 사용하여 사용자 기대를 설정합니다.
- 각 단계에서 사용자 입력을 확인하여 오류를 조기에 발견하고 오류를 줄일 수 있습니다.
- 사용자가 뒤로 이동하여 데이터 손실 없이 이전 정보를 검토하거나 편집할 수 있습니다.

**일반적인 사용 사례**

- **보험 청구:** 문제 세부 정보, 증빙 제출, 개인 정보 및 검토에 대한 단계입니다.
- **계정 설정:** 기본 정보, 환경 설정, 보안 설정 및 확인을 위한 단계입니다.
- **주문 프로세스:** 제품 선택, 배송 정보, 결제 세부 사항 및 주문 요약에 대한 단계입니다.

### 아코디언 레이아웃

아코디언 레이아웃은 콘텐츠를 축소 가능한 섹션으로 구성하여 공간을 절약하므로 옵션 또는 보조 정보에 이상적입니다. 이 레이아웃은 논리적으로 그룹화할 수 있고 한 번에 모두 표시할 필요가 없는 콘텐츠가 있는 양식에 특히 유용합니다.

![아코디언 레이아웃 예제](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**응답형 동작**

- **모바일 성능:** 관련 섹션만 확장되어 스크롤의 필요성을 줄이고 로드 시간을 개선합니다.
- **터치에 적합한 머리글:** 섹션 머리글은 탭하거나 확장하기 쉬우며 모바일 장치에서 자연스러운 제스처를 지원합니다.
- **매끄러운 애니메이션:** 섹션 확장 및 축소에서는 사용자 상호 작용에 대한 시각적 피드백을 제공합니다.
- **공간 효율성:** 축소된 섹션은 세로 공간을 최소화하므로 모든 장치에서 양식을 쉽게 탐색할 수 있습니다.

**구현 방법**

1. 양식에 [아코디언 구성 요소](https://experienceleague.adobe.com/ko/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion)를 추가합니다.
2. 각 아코디언 섹션 내에서 관련된 선택적 콘텐츠 또는 보조 콘텐츠를 그룹화합니다.
3. 각 섹션에 대해 명확하고 설명적인 헤더를 사용하여 내부에 포함된 정보를 이해할 수 있습니다.
4. 중요도 및 사용자 요구 사항에 따라 각 섹션에 적절한 기본 열기 또는 닫기 상태를 설정합니다.

**모바일 이점**

- 사용하지 않는 섹션을 축소하여 스크롤을 줄여 한 번에 한 섹션에 집중할 수 있습니다.
- 터치 친화적 상호 작용은 자연스러운 확장/축소 제스처를 지원한다.
- 활성 콘텐츠만 표시되므로 더 빠르게 로드할 수 있습니다.
- 사용자가 지정된 시간에 필요한 정보만 볼 수 있으므로 포커스가 개선되었습니다.

**모범 사례**

- 섹션을 확장하기 전에 예상할 내용을 사용자가 알 수 있도록 섹션 헤더 지우기 를 사용합니다.
- 이해를 돕기 위해 각 섹션 내에서 관련 콘텐츠를 논리적으로 그룹화합니다.
- 즉각적인 주의가 필요한 경우 확장하도록 중요 섹션을 설정합니다.
- 사용자가 확장할 섹션을 결정하는 데 도움이 되는 간단한 섹션 미리보기 또는 요약을 제공합니다.

**일반적인 사용 사례**

- **제품 구성:** 기본 옵션, 고급 설정, 액세서리 및 지원 섹션.
- 계정, 청구, 기술 및 일반 질문에 대한 **FAQ Forms:** 그룹화.
- **설정 Forms:** 개인 정보, 알림, 모양 및 고급 옵션에 대한 섹션.

### 장치 유형별 우수 사례

+++모바일 최적화(320px-767px)

- 가독성과 사용 편의성을 극대화하려면 모든 양식 콘텐츠에 대해 단일 열 레이아웃을 사용하십시오.
- 신뢰할 수 있는 터치 상호 작용을 위해 모든 버튼과 대화형 요소의 높이가 최소 44px인지 확인하십시오.
- 뒤로 및 다음 버튼이 표시되는 명확하고 간단한 탐색을 제공합니다.
- 긴 양식을 분할하여 각 섹션 내에서 스크롤할 필요성을 최소화합니다.
- 첫 번째 입력 필드에 자동으로 초점을 맞추어 모바일 키보드를 확인합니다.

**필드 지침:**

- 텍스트 필드는 터치 입력을 위한 충분한 패딩과 함께 화면의 전체 너비에 걸쳐 있어야 합니다.
- 최적의 모바일 유용성을 위해 기본 드롭다운/선택 요소를 사용합니다.
- 일관된 모바일 경험을 위해 기본 날짜 선택기를 구현합니다.
- 쉽게 액세스할 수 있도록 파일 업로드 영역을 크게 만들고 레이블을 명확히 지정합니다.

+++

+++태블릿 최적화(768px-1199px)

**레이아웃 및 유용성:**

- 관련 필드에 대해 2열 레이아웃을 활용하여 화면 공간을 늘릴 수 있습니다.
- 세로 및 가로 방향 모두에서 양식 모양 및 유용성을 테스트합니다.

- 명확한 시각적 계층과 가독성을 유지하면서 콘텐츠 영역 크기를 늘립니다.

+++

+++데스크탑 최적화(1200px+)

**고급 기능 및 레이아웃:**

- 대화형 요소에 대한 가리키기 상태 및 시각적 피드백을 구현합니다.
- 복잡한 양식에 대한 명확하고 자세한 오류 메시지를 통해 고급 유효성 검사를 제공합니다.

## 포괄적인 문제 해결

### 레이아웃 문제

+++모바일에서 양식 레이아웃 나누기

**가능한 원인:**

- 작은 화면에 적응하지 않는 고정 너비 요소
- 모바일 스타일을 무시하는 데스크톱 우선 CSS
- 컨테이너를 넘쳐나는 이미지 또는 콘텐츠

**해결 방법:**

- 모든 이미지와 컨테이너가 상대 크기 또는 백분율 기반 크기를 사용하는지 확인하십시오.
- 모바일 우선 CSS 접근 방식으로 시작하여 더 큰 화면을 위한 개선 사항에 대한 레이어를 적용하십시오.
- 장치 에뮬레이터와 실제 장치를 모두 사용하여 양식을 테스트합니다.
- 고정된 치수를 사용하지 마십시오. 유연한 레이아웃을 사용하십시오.

+++

+++터치 대상이 너무 작음

**가능한 원인:**

- 44px x 44px보다 작은 단추 또는 링크
- 너무 가까이 배치된 대화형 요소
- 사용자 지정 CSS로 기본 터치 대상 크기 축소

**해결 방법:**

- 모든 대화형 요소가 44px x 44px 이상인지 확인합니다.
- 단추, 링크 및 기타 컨트롤 사이에 적절한 간격을 추가합니다.
- 마우스뿐만 아니라 실제 터치 장치로 테스트합니다.
- 필요에 따라 접근성을 위해 터치 대상 영역을 확장합니다.

+++

+++컨텐츠 오버플로 문제

**가능한 원인:**

- 줄 바꿈하지 않는 긴 텍스트 또는 레이블
- 고정된 너비의 컨테이너
- 반응형으로 확장되지 않는 이미지

**해결 방법:**

- 모든 레이블 및 콘텐츠에 대해 텍스트 줄바꿈을 활성화합니다.
- 컨테이너를 통해 크기가 조절되는 응답형 이미지를 사용합니다.
- 다양한 콘텐츠 길이에 맞게 유연한 레이아웃을 디자인할 수 있습니다.
- 적응성을 보장하기 위해 짧은 콘텐츠와 긴 콘텐츠로 테스트합니다.

+++

### 성능 문제

+++모바일 로드 속도 느림

**가능한 원인:**

- 최적화되지 않은 대형 이미지
- 과다 또는 과다 JavaScript
- 동시에 너무 많은 양식 필드를 로드함

**해결 방법:**

- 모바일용 이미지를 최적화하고 적절한 파일 형식을 사용합니다.
- 중요하지 않은 콘텐츠를 지연 또는 지연 로드합니다.
- 서드파티 스크립트 및 위젯의 사용을 최소화합니다.
- 필요한 항목만 로드하도록 양식 필드를 간소화합니다.

+++

### 테스트 및 유효성 검사 문제

+++에뮬레이터와 실제 장치의 차이점

**가능한 원인:**

- 브라우저 렌더링 엔진의 차이점
- 마우스로 정확하게 시뮬레이션되지 않은 터치 상호 작용
- 네트워크 속도 불일치

**해결 방법:**

- 에뮬레이터와 함께 항상 실제 디바이스에서 테스트하십시오.
- 포괄적인 테스트를 위해 여러 브라우저 및 디바이스를 사용하십시오.
- 다양한 네트워크 속도를 시뮬레이션하여 성능 병목 현상을 식별합니다.
- 타겟 대상의 실제 사용자로부터 피드백을 수집합니다.
