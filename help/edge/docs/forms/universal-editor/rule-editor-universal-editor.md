# 범용 편집기의 Dynamic Forms용 규칙 편집기

규칙 편집기를 사용하면 작성자가 코드를 작성하지 않고도 정적 양식을 반응형 지능형 경험으로 전환할 수 있습니다. 조건부로 필드를 표시하고, 계산을 수행하고, 데이터의 유효성을 검사하고, 흐름을 통해 사용자를 안내하고, 사람 유형에 맞게 조정하는 비즈니스 논리를 통합할 수 있습니다.

## 배울 내용

이 안내서를 마치면 다음 작업을 수행할 수 있습니다.

- 규칙의 작동 방식과 다양한 규칙 유형을 사용할 시기를 이해합니다.
- 범용 편집기에서 규칙 편집기 활성화 및 액세스
- 필드를 동적으로 표시하거나 숨기기 위한 조건부 논리 만들기
- 자동화된 계산 및 데이터 유효성 검사 구현
- 복잡한 비즈니스 규칙에 대한 사용자 정의 함수 구축
- 성능, 유지 관리 및 UX에 대한 모범 사례 적용

## 규칙 편집기를 사용하는 이유

- **조건부 논리**: 노이즈 및 인지 부하를 줄이는 데 필요한 경우에만 관련 필드를 표시합니다.
- **동적 계산**: 사용자 유형에 따라 값(합계, 비율, 세금)을 자동으로 계산합니다.
- **데이터 유효성 검사**: 실시간 확인 및 메시지 지우기를 통해 오류를 조기에 방지합니다.
- **안내 환경**: 논리적 단계(마법사, 분기)를 안내합니다.
- **코드 작성 안 함**: 시각적 인터페이스를 통해 강력한 동작을 구성합니다.

일반적인 시나리오에는 세금 계산자, 대출 및 프리미엄 추정자, 자격 흐름, 다단계 신청, 조건부 질문이 있는 조사가 포함됩니다.

## 규칙 작동 방식

규칙은 조건이 충족될 때 발생하는 작업을 정의합니다. 개념적으로, 규칙에는 두 가지 부분이 있습니다.

- **조건**: true 또는 false로 평가되는 문입니다.
   - 예: &quot;소득 > 50,000&quot;, &quot;수혜범위 = &#39;예&#39;,&quot; &quot;필드가 비어 있음&quot;
- **작업**: 조건이 true일 때(선택적으로 false일 때) 발생하는 작업.
   - 예: 필드 표시/숨기기, 값 설정/지우기, 입력 유효성 검사, 단추 활성화/비활성화

+++ 규칙 논리 패턴

- **조건 → 동작(When/Then)**

  ```text
  WHEN Gross Salary > 50000
  THEN Show "Additional Deduction"
  ```

  조건부 가시성 및 점진적 공개에 가장 적합합니다.

- **동작 ← 조건(If/Only) 설정**

  ```text
  SET Taxable Income = Gross Salary - Deductions
  IF Deductions are applicable
  ```

  계산 및 데이터 변환에 가장 적합합니다.

- **→ 경우 →(대체 작업)**

  ```text
  IF Income > 50000
  THEN Show "High Income" fields
  ELSE Show "Standard Income" fields
  ```

  분기 논리 및 상호 배타적인 흐름에 가장 적합합니다.

+++

+++ 실제 사례

- **조건**: &quot;총 급여가 $50,000를 초과합니다.&quot;
- **기본 작업**: &quot;추가 공제&quot; 표시
- **대체 작업**: &quot;추가 공제&quot; 숨기기
- **결과**: 사용자에게 적용되는 필드만 표시됨

+++

## 사전 요구 사항


+++ 액세스 요구 사항

**필수 권한 및 설정**:

- **AEM as a Cloud Service**: 양식 편집 권한으로 액세스 권한 작성
- **유니버설 편집기**: 환경에 설치 및 구성됨
- **규칙 편집기 확장**: [Extension Manager](/help/implementing/developing/extending/extension-manager.md)을 통해 사용
- **양식 편집 권한**: 유니버설 편집기에서 양식 구성 요소를 만들고 수정하는 기능

**확인 단계**:

1. AEM Sites 콘솔에서 범용 편집기에 액세스할 수 있는지 확인합니다.
2. 양식 구성 요소를 만들고 편집할 수 있는지 확인
3. 양식 구성 요소를 선택할 때 규칙 편집기 아이콘 ![edit-rules](/help/forms/assets/edit-rules-icon.svg)이(가) 표시되는지 확인하십시오.

+++

+++ 기술 요구 사항

**필요한 지식과 기술**:

- **유니버설 편집기 숙련도**: 텍스트 입력, 드롭다운 및 기본 필드 속성을 사용하여 양식을 만드는 경험
- **비즈니스 논리 이해**: 특정 사용 사례에 대한 조건부 요구 사항 및 유효성 검사 규칙을 정의하는 기능
- **양식 구성 요소에 익숙함**: 필드 형식(텍스트, 숫자, 드롭다운), 속성(필수, 표시, 읽기 전용) 및 양식 구조에 대한 지식입니다.

**고급 사용의 옵션**:

- **JavaScript 기본 사항**: 사용자 지정 함수(데이터 형식, 함수, 기본 구문)를 만드는 데만 필요합니다.
- **JSON 이해**: 복잡한 데이터 조작 및 API 통합에 유용합니다.

**평가 질문**:

- 범용 편집기에서 텍스트 입력 및 제출 버튼을 사용하여 기본 양식을 만들 수 있습니까?
- 비즈니스 컨텍스트에서 필수 필드와 선택 필드의 경우를 알고 계십니까?
- 사용 사례에서 조건부 가시성이 필요한 양식 요소를 식별할 수 있습니까?

+++

+++ 규칙 편집기 확장 활성화

**중요**: 규칙 편집기 확장은 유니버설 편집기 환경에서 기본적으로 활성화되지 않습니다.

**활성화 단계**:

1. AEM 환경의 [Extension Manager](/help/implementing/developing/extending/extension-manager.md)(으)로 이동
2. 사용 가능한 확장 목록에서 &quot;규칙 편집기&quot; 확장을 찾습니다
3. **활성화**&#x200B;를 클릭하고 활성화를 확인합니다
4. 시스템이 새로 고쳐질 때까지 대기(1~2분 정도 소요될 수 있음)

**확인**:

- 활성화한 후 양식 구성 요소를 선택하면 규칙 편집기 아이콘이 나타납니다. ![edit-rules](/help/forms/assets/edit-rules-icon.svg)

![유니버설 편집기 규칙 편집기](/help/edge/docs/forms/assets/universal-editor-rule-editor.png)
그림: 양식 구성 요소를 선택하면 규칙 편집기 아이콘이 표시됩니다

규칙 편집기를 열려면

1. 범용 편집기에서 양식 구성 요소를 선택합니다.
2. 규칙 편집기 아이콘을 클릭합니다.
3. 규칙 편집기가 사이드 패널에서 열립니다.

![규칙 편집기 사용자 인터페이스](/help/edge/docs/forms/assets/rule-editor-for-field.png)
그림: 구성 요소 규칙을 편집하기 위한 규칙 편집기 인터페이스

>[!NOTE]
>
> 이 문서에서 &quot;양식 구성 요소&quot;와 &quot;양식 개체&quot;는 동일한 요소(예: 입력, 단추, 패널)를 참조합니다.

## 규칙 편집기 인터페이스 개요

![규칙 편집기 사용자 인터페이스](/help/edge/docs/forms/assets/rule-editor-interface.png)
그림: 번호가 매겨진 구성 요소가 있는 전체 규칙 편집기 인터페이스

- **구성 요소 제목 및 규칙 유형**: 선택한 구성 요소와 활성 규칙 유형을 확인합니다.
- **양식 개체 및 함수 패널**:
   - 양식 개체: 규칙에서 참조할 필드 및 컨테이너의 계층적 보기
   - 함수: 내장 수학, 문자열, 날짜 및 유효성 검사 도우미
- **패널 전환**: 작업 영역을 늘리려면 개체 및 함수 패널을 표시하거나 숨깁니다.
- **시각적 규칙 빌더**: 드롭다운 기반 규칙 작성기 드래그 앤 드롭
- **컨트롤**: 완료(저장), 취소(취소) 저장하기 전에 항상 규칙을 테스트하십시오.

+++

+++ 기존 규칙 관리

구성 요소에 규칙이 이미 있는 경우 다음 작업을 수행할 수 있습니다.

- **보기**: 규칙 요약 및 논리 보기
- **편집**: 조건 및 작업 수정
- **순서 바꾸기**: 실행 순서 변경(위에서 아래로)
- **활성화/비활성화**: 테스트를 위한 규칙 전환
- **삭제**: 규칙을 안전하게 제거합니다.

>[!TIP]
>
> 일반적인 규칙보다 특정 규칙을 우선시하십시오. 실행은 하향식입니다.

+++

## 사용 가능한 규칙 유형

사용자의 의도에 가장 적합한 규칙 유형을 선택합니다.

+++ 조건부 논리

- **When**: 복잡한 조건부 동작에 대한 기본 규칙(Condition → Action ± Else)
- **숨기기/표시**: 조건에 따라 가시성을 제어합니다(점진적 공개).
- **활성화/비활성화**: 필드의 대화형 여부를 제어합니다(예: 필수 필드가 유효할 때까지 제출 비활성화).

+++

+++ 데이터 조작

- **값 설정**: 값(예: 날짜, 합계, 복사본) 자동 채우기
- **값 지우기**: 조건이 변경되면 데이터를 제거합니다.
- **형식**: 저장된 값을 변경하지 않고 표시 형식(통화, 휴대폰, 날짜)을 변환합니다.

+++

+++ 유효성 검사

- **유효성 검사**: 교차 필드 검사 및 비즈니스 규칙을 포함한 사용자 지정 유효성 검사 논리

+++

+++ 계산

- **수학 식**: 실시간으로 값 계산(합계, 세금, 비율)

+++

+++ 사용자 인터페이스

- **포커스 설정**: 포커스를 특정 필드로 이동합니다(제한적으로 사용).
- **속성 설정**: 구성 요소 속성(자리 표시자, 옵션 등)을 동적으로 수정합니다.

+++

+++ 양식 컨트롤

- **양식 제출**: 유효성 검사를 통과한 후에만 프로그래밍 방식으로 양식을 제출합니다.
- **양식 재설정**: 지우고 초기 상태로 재설정(사용 전 확인)
- **양식 저장**: 나중에 사용할 수 있도록 초안으로 저장(긴 양식, 다중 세션)

+++

+++ 고급

- **서비스 호출**: 외부 API/서비스 호출(로드 및 오류 처리)
- **인스턴스 추가/제거**: 반복 가능한 섹션(예: 종속 항목, 주소) 관리
- **이동**: 다른 양식/페이지로 이동(탐색 전에 데이터 유지)
- **패널 간 탐색**: 컨트롤 마법사 단계 탐색 및 건너뛰기
- **이벤트 발송**: 통합 또는 분석에 대한 사용자 지정 이벤트 트리거

+++

## 단계별 자습서: 스마트 세금 계산기 구축

+++ 자습서 개요

이 예에서는 조건부 가시성 및 자동 계산을 보여 줍니다.

![양식 필드 가시성에 대한 When-Then 논리를 사용하여 조건부 규칙을 만드는 것을 보여 주는 규칙 편집기 인터페이스의 스크린샷](/help/edge/docs/forms/assets/rule-editor-1.png)
그림: 지능형 조건부 필드가 있는 세금 계산 양식

다음과 같은 양식을 작성합니다.

1. 관련 필드를 표시하여 사용자 입력에 맞게 조정
2. 실시간으로 값 계산
3. 정확도를 개선하기 위해 데이터 유효성 검사

+++

+++ 양식 구조

| 필드 이름 | 유형 | 용도 | 비헤이비어 |
|-------------------------|---------------|--------------------------------|-----------------------------------------|
| 총 급여 | 숫자 입력 | 사용자의 연간 수입 | 조건부 논리 트리거 |
| 추가 공제 | 숫자 입력 | 추가 공제(해당되는 경우) | 급여가 $50,000인 경우에만 표시 |
| 과세 소득 | 숫자 입력 | 계산된 값 | 읽기 전용, 변경 시 업데이트 |
| 납부세 | 숫자 입력 | 계산된 값 | 읽기 전용, 정액 비율로 계산 |

+++

+++ 비즈니스 논리

- **규칙 1: 조건부 표시**

  ```text
  WHEN Gross Salary > 50,000
  THEN Show "Additional Deduction"
  ELSE Hide "Additional Deduction"
  ```

- **규칙 2: 과세 소득 계산**

  ```text
  SET Taxable Income = Gross Salary - Additional Deduction
  (Only when Additional Deduction applies)
  ```

- **규칙 3: 납부세액 계산**

  ```text
  SET Tax Payable = Taxable Income × 10%
  (Simplified flat rate)
  ```

+++

+++ 1단계: 기본 양식 만들기

**목표**: 모든 필드 및 초기 설정으로 기본 양식을 빌드합니다.

1. **유니버설 편집기 열기**:
   - AEM Sites 콘솔로 이동하여 페이지를 선택하고 **편집**&#x200B;을 클릭합니다.
   - [유니버설 편집기](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/implementing/developing/universal-editor/introduction.html)가 제대로 구성되어 있는지 확인하십시오.

2. **이 순서로 양식 구성 요소 추가**:
   - 제목(H2): &quot;세금 계산 양식&quot;
   - 숫자 입력: &quot;총 임금&quot;(필수: 예, 자리 표시자: &quot;연간 임금 입력&quot;)
   - 숫자 입력: &quot;추가 공제&quot;(필수: 아니오, 자리 표시자: &quot;추가 공제 입력&quot;)
   - 숫자 입력: &quot;과세 소득&quot;(읽기 전용: 예)
   - 번호 입력: &quot;지급 세금&quot;(읽기 전용: 예)
   - 제출 단추: &quot;세금 계산&quot;

3. **초기 필드 속성 구성**:
   - &quot;추가 공제&quot; 숨기기(속성 패널에서 표시: 아니요 설정)
   - &quot;과세 소득&quot; 및 &quot;납부세액&quot;을 읽기 전용으로 설정: 예

+++ 2단계: 조건부 가시성 규칙 추가

**목표**: 총 급여가 $50,000를 초과하는 경우에만 &quot;추가 공제&quot; 필드를 표시합니다.

1. **총 급여 필드 선택** 및 규칙 편집기 아이콘 ![편집 규칙](/help/forms/assets/edit-rules-icon.svg) 클릭
2. **새 규칙 만들기**:
   - **만들기**&#x200B;를 클릭합니다.
   - 규칙 유형을 &quot;Set Value Of&quot;에서 **&quot;When&quot;**(으)로 변경
3. **조건 구성**:
   - 드롭다운에서 **&quot;이(가) &quot;**&quot;보다 큼 선택
   - 숫자 필드에 `50000`을(를) 입력하십시오.
4. **Then 작업 설정**:
   - 작업 선택 드롭다운에서 **&quot;표시&quot;** 선택
   - 양식 개체에서 **&quot;추가 공제&quot;** 필드를 드래그하거나 선택하십시오.
5. **Else 작업 추가**:
   - **&quot;기타 섹션 추가&quot;** 클릭
   - 작업 선택 드롭다운에서 **&quot;숨기기&quot;** 선택
   - **&quot;추가 공제&quot;** 필드 선택
6. **규칙 저장**: **완료** 클릭

>[!NOTE]
>
> 대체 방법: &quot;총 임금&quot;에 대한 When 규칙 대신 &quot;추가 공제&quot; 필드에 직접 표시/숨기기 규칙을 생성하여 동일한 결과를 얻을 수 있습니다.

+++ 3단계: 계산 규칙 추가

**목표**: 사용자 입력에 따라 &quot;과세 소득&quot; 및 &quot;납부세액&quot;을 자동으로 계산합니다.

**과세 소득 계산 구성**:

1. **&quot;과세 소득&quot; 필드 선택** 및 규칙 편집기 열기
2. **수학 식 만들기**:
   - **만들기**→ 클릭하고 **&quot;수학 식&quot;** 선택
   - 빌드 식: **총 급여 − 추가 공제**
   - 첫 번째 필드로 &quot;총 임금&quot;을 드래그합니다.
   - **&quot;빼기&quot;** 연산자 선택
   - 두 번째 필드로 &quot;추가 공제&quot; 드래그
3. **저장**: **완료** 클릭

**납부세 계산 구성**:

1. **&quot;납부세&quot; 필드 선택** 및 규칙 편집기 열기
2. **수학 식 만들기**:
   - **만들기**→ 클릭하고 **&quot;수학 식&quot;** 선택
   - 빌드 식: **과세 소득 × 10 ÷ 100**
   - 첫 번째 필드로 &quot;과세 소득&quot; 드래그
   - **&quot;곱하기&quot;** 연산자 선택
   - `10`을(를) 숫자로 입력
   - **&quot;식 확장&quot;** 클릭
   - **&quot;나누기&quot;** 연산자 선택
   - `100`을(를) 숫자로 입력
3. **저장**: **완료** 클릭

+++ 4단계: 양식 테스트

**전체 흐름을 테스트하여 구현 확인**:

1. **양식 미리 보기**: 유니버설 편집기에서 미리 보기 모드를 클릭합니다.
2. **조건부 논리 테스트**:

   - 총 급여 = `60000`을(를) → &quot;추가 공제&quot;가 표시됩니다.
3. **계산 테스트**:
   - 총 급여 = `60000`인 경우 추가 공제 = `5000`을(를) 입력하십시오.

   - 납부할 세액 확인 = `5500`(55000 × 10%)

- **외부 통합**: 내부 API 또는 서드파티 서비스 호출(제한 사항 있음)

- **성능**: 복잡한 규칙 체인과 비교하여 최적화된 JavaScript 실행
- **유연성**: 표준 규칙에서 다루지 않는 경계 사례 및 복잡한 시나리오를 처리합니다.

+++ 사용자 정의 함수 만들기 및 구현

**파일 위치**: 모든 사용자 지정 함수는 Edge Delivery Services 프로젝트의 `/blocks/form/functions.js`에 정의되어야 합니다.

**개발 워크플로**:

   - 수사적 작업 지향 함수 이름 사용

2. **구현**

3. **설명서**

   - 제한 사항 또는 종속성 문서화

**예제 구현**:

```javascript
/**
 * Concatenates first and last name with proper formatting
 * @name getFullName
 * @description Combines first and last name, handles edge cases like missing values
 * @param {string} firstName - The person's first name
 * @param {string} lastName - The person's last name  
 * @returns {string} Formatted full name or empty string if both inputs are invalid
 */
function getFullName(firstName, lastName) {
  // Handle null, undefined, or empty string inputs
  const first = (firstName || '').toString().trim();
  const last = (lastName || '').toString().trim();
  
  return `${first} ${last}`.trim();
}

/**
 * Calculates the number of days between two dates
 * @name days
 * @description Computes absolute difference in days, handles various date input formats
 * @param {Date|string} endDate - End date (Date object or ISO string)
 * @param {Date|string} startDate - Start date (Date object or ISO string)
 * @returns {number} Number of days between dates, 0 if inputs are invalid
 */
function days(endDate, startDate) {
  // Convert string inputs to Date objects
  const start = typeof startDate === 'string' ? new Date(startDate) : startDate;
  const end = typeof endDate === 'string' ? new Date(endDate) : endDate;

  // Validate date objects
  if (Number.isNaN(start.getTime()) || Number.isNaN(end.getTime())) {
    return 0;
  }

  // Calculate absolute difference in milliseconds, then convert to days
  const diffInMs = Math.abs(end.getTime() - start.getTime());
  return Math.floor(diffInMs / (1000 * 60 * 60 * 24));
}

// Export functions for use in Rule Editor
export { getFullName, days };
```

## 요약

범용 편집기의 규칙 편집기는 정적 양식을 사용자 입력에 실시간으로 적응하는 지능적이고 반응형 경험으로 변환합니다. 조건부 논리, 자동화된 계산 및 사용자 지정 비즈니스 규칙을 활용하여 애플리케이션 코드를 작성하지 않고도 정교한 양식 워크플로우를 만들 수 있습니다.

**학습한 주요 기능**:

- **조건부 논리**: 사용자 입력을 기반으로 필드를 표시하고 숨겨 포커스가 설정된 관련 경험을 만듭니다.
- **동적 계산**: 사용자가 양식과 상호 작용할 때 값(세금, 합계, 비율)을 자동으로 계산합니다.
- **데이터 유효성 검사**: 실행 가능한 명확한 피드백 메시지로 실시간 유효성 검사를 구현합니다.
- **사용자 지정 함수**: 복잡한 비즈니스 논리 및 통합을 위해 JavaScript을 사용하여 기능을 확장합니다.
- **성능 최적화**: 유지 관리 가능하고 효율적인 규칙 개발을 위한 모범 사례 적용

**전달된 값**:

- **향상된 사용자 경험**: 점진적 공개 및 지능형 양식 흐름을 통해 인지 부하 감소
- **줄어든 오류**: 실시간 유효성 검사 및 안내 입력을 통해 잘못된 제출을 방지합니다.
- **효율성 향상**: 계산 및 데이터 입력을 자동화하여 사용자 노력을 최소화합니다.
- **유지 관리 가능한 솔루션**: 조직 전체에 걸쳐 확장되는 재사용 가능한 잘 문서화된 규칙을 만듭니다.

**비즈니스 영향**:

Forms은 데이터 수집, 잠재 고객 검증 및 사용자 참여를 위한 강력한 도구가 됩니다. 규칙 편집기를 사용하면 기술 전문가가 아닌 작성자가 정교한 비즈니스 논리를 구현할 수 있으므로 개발 비용을 절감하는 동시에 양식 완료율과 데이터 품질을 향상시킬 수 있습니다.
