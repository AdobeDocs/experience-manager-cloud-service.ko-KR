# 감사 메시지 및 리디렉션 URL 구성

사후 제출 경험은 사용자 만족도와 양식 완료율에 상당한 영향을 미칩니다. Adobe의 유니버설 편집기는 개인화된 감사 메시지나 특정 페이지로의 전략적 리디렉션 여부에 관계없이 양식을 제출한 후 사용자에게 표시되는 내용을 구성하는 포괄적인 옵션을 제공합니다.

이 문서에서는 양식 제출의 효과를 극대화하기 위한 기술 고려 사항, 모범 사례 및 사용자 경험 지침을 포함하여 감사 메시지와 리디렉션 URL을 모두 구현하는 방법에 대한 자세한 지침을 제공합니다.

## 사전 요구 사항

사후 제출 경험을 구성하기 전에 다음을 확인하십시오.

**기술 설정:**

- 적절한 권한으로 유니버설 편집기에 액세스
- 범용 편집기에서 만든 기존 적응형 양식
- 조직의 리디렉션 URL 요구 사항 이해

**계획 고려 사항:**

- **메시지 전략**: 감사 메시지에 포함할 색조, 길이 및 특정 정보를 정의합니다.
- **리디렉션 전략**: 대상 페이지를 식별하고 사후 양식 완료 환경에 최적화되었는지 확인합니다.
- **Analytics 통합**: 감사 메시지 또는 리디렉션 대상으로 사용자 상호 작용을 추적하는 방법을 계획합니다.

## 감사 인사 메시지 구성

감사 메시지는 성공적인 양식 제출을 즉시 승인하며 개인화된 콘텐츠, 다음 단계 또는 사용자의 제출과 관련된 중요한 정보를 포함할 수 있습니다.

### 감사 인사 메시지 사용 시기

감사 인사 메시지는 다음과 같은 경우에 가장 잘 작동합니다.

- **단순 확인**: 사용자는 추가 탐색 요구 사항 없이 확인이 필요합니다.
- **지침 콘텐츠**: 특정 다음 단계 또는 중요한 정보를 제공해야 합니다.
- **브랜드 일관성**: 조직의 커뮤니케이션 스타일에 맞게 메시지를 만들 수 있습니다.
- **단일 페이지 경험**: 워크플로우 연속성을 위해 사용자는 현재 페이지에 남아 있어야 합니다.

### 구현 단계

**1. 양식 속성 액세스**

범용 편집기에서 적응형 양식을 열고 도구 모음에서 **양식 속성 편집** 아이콘을 클릭합니다. 그러면 포괄적인 양식 속성 대화 상자가 열립니다.

**2. 감사 인사 구성**(으)로 이동

양식 속성 대화 상자에서 **감사합니다** 탭을 선택하여 사후 제출 구성 옵션에 액세스합니다.

**3. 메시지 표시 구성**

사용 가능한 옵션에서 **메시지 표시**&#x200B;를 선택합니다. 리치 텍스트 기능이 포함된 메시지 콘텐츠 편집기가 활성화됩니다.

**4. 메시지 콘텐츠 만들기**

**메시지 콘텐츠** 필드에서 서식 있는 텍스트 편집기를 사용하여 감사 메시지를 작성하십시오. 편집기는 다음을 지원합니다.

- **텍스트 서식**: 굵게, 기울임꼴, 밑줄 및 색상 옵션
- **목록**: 정보를 구성할 글머리 기호 및 번호 매기기 목록
- **링크**: 관련 리소스 또는 다음 단계로 가는 직접 링크
- **전체 화면 편집**: 더 큰 편집 작업 영역의 확장 아이콘을 클릭합니다.

### 기술 고려 사항

**메시지 표시 동작:**

- 메시지는 양식 제출 직후 모달 오버레이에 나타납니다
- 콘텐츠는 HTML 형식을 지원하고 응답형 디자인을 유지 관리합니다.
- 메시지는 사용자가 해제하거나 자동 닫기 타이머로 구성할 수 있습니다.

**콘텐츠 지침:**

- 필요한 정보를 제공하면서 메시지를 간결하게 보관
- 해당되는 경우 명확한 다음 단계 포함
- 참조 번호 또는 확인 세부 정보를 포함하는 것이 좋습니다.
- 모바일 친화적 형식 지정

## 리디렉션 URL 구성

리디렉션 URL은 양식 제출 후 사용자를 자동으로 특정 페이지로 안내하여 기존 워크플로우와 원활하게 통합하거나 사용자를 관련 콘텐츠로 안내합니다.

### 리디렉션 URL 사용 시기

리디렉션 URL은 다음에 최적:

- **워크플로우 통합**: 대시보드, 계정 페이지 또는 프로세스의 다음 단계로 사용자 지정
- **컨텐츠 전달**: 양식 응답을 기반으로 관련 제품, 서비스 또는 정보 표시
- **Analytics 추적**: 특정 추적 구현이 있는 페이지로 이동
- **여러 단계 프로세스**: 사용자를 복잡한 워크플로의 다음 단계로 이동

### 구현 단계

**1. 양식 속성 액세스**

범용 편집기에서 적응형 양식을 열고 **양식 속성 편집** 아이콘을 클릭하여 양식 구성 대화 상자를 엽니다.

**2. 감사 인사 구성**(으)로 이동

양식 속성 대화 상자에서 **감사합니다** 탭을 선택하여 리디렉션 구성 옵션에 액세스합니다.

**3. 리디렉션 기능 사용**

사용 가능한 사후 제출 옵션에서 **URL로 리디렉션**&#x200B;을 선택합니다.

**4. 대상 URL** 구성

제공된 필드에 타겟 URL을 입력합니다. 시스템에서는 유연한 구현을 위해 여러 URL 형식을 지원합니다.

### URL 구성 옵션

**절대 URL**

프로토콜 및 도메인을 포함한 전체 웹 주소:

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**상대 경로**

현재 도메인을 기준으로 하는 경로:

    /감사
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sites 페이지 참조**

AEM Sites 구현 내의 다른 페이지에 대한 참조:

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 기술 고려 사항

**리디렉션 동작:**

- 양식 제출 직후 리디렉션이 발생합니다
- 브라우저 기록에 올바른 백 버튼 기능을 위한 리디렉션이 포함됩니다
- 리디렉션 타이밍은 선택적 지연으로 구성할 수 있습니다.

**URL 유효성 검사:**

- 구성을 허용하기 전에 URL 형식의 유효성을 검사합니다.
- 상대 URL은 현재 도메인에 대해 확인됩니다.
- 필요한 경우 외부 URL을 사용하려면 적절한 CORS 구성이 필요합니다

## 우수 사례 및 권장 사항

### 사용자 경험 지침

**메시지 최적화:**

- **명확성 우선**: 사용자가 제출이 성공했음을 즉시 인식하도록 합니다.
- **값 추가**: 사용자가 다음 단계를 수행하는 데 도움이 되는 정보를 제공합니다.
- **일관된 브랜딩**: 조직의 음성 및 시각적 스타일 유지
- **모바일 고려 사항**: 다양한 화면 크기에서 메시지 테스트

**리디렉션 최적화:**

- **페이지 최적화**: 리디렉션 대상이 사후 양식 방문자에 맞게 최적화되었는지 확인
- **성능 로드**: 사용자 환경을 유지하려면 리디렉션 페이지가 빠르게 로드되는지 확인하십시오.
- **콘텐츠 관련성**: 리디렉션 콘텐츠가 양식 컨텍스트와 관련이 있는지 확인합니다.

### 보안 고려 사항

**URL 유효성 검사:**

- 악성 리디렉션을 방지하기 위해 리디렉션 URL에 대한 적절한 유효성 검사 구현
- 허용된 리디렉션 도메인에 대한 허용 목록 접근 방식 고려
- 비정상적인 활동에 대한 리디렉션 패턴 모니터링

**콘텐츠 보안:**

- 스크립트 삽입을 방지하기 위해 감사 메시지 콘텐츠 삭제
- 리치 텍스트 컨텐츠에 대한 적절한 컨텐츠 보안 정책 구현
- 리디렉션 대상에 대한 정기적인 보안 검토

### 분석 및 추적

**구현 고려 사항:**

- **목표 추적**: 감사 메시지 보기와 리디렉션 완료 모두에 대한 분석 목표를 설정합니다.
- **사용자 여정 매핑**: 사용자가 제출 후 경험과 상호 작용하는 방법을 추적합니다.
- **전환 최적화**: A/B에서 다른 감사 메시지와 리디렉션 대상을 테스트합니다.

**측정 전략:**

- 해고 전 감사 메시지에서 보낸 시간 모니터링
- 감사 인사 메시지 내의 링크에 대한 클릭스루 비율 추적
- 리디렉션 대상 페이지에서 사용자 비헤이비어 분석

## 유효성 검사 체크포인트

사후 제출 경험을 구성한 후:

**구성 확인:**

- 양식 속성에 선택한 감사 인사 옵션이 올바르게 표시됨
- 메시지 콘텐츠가 미리보기 모드에서 제대로 표시됨
- 리디렉션 URL의 형식이 적절하며 액세스 가능
- 메시지 내의 모든 링크가 올바르게 작동합니다.

**사용자 환경 테스트:**

- 테스트 양식을 제출하여 적절한 감사 메시지 표시 확인
- 여러 브라우저에서 리디렉션 기능 테스트
- 감사 메시지의 모바일 응답성 확인
- 리디렉션 대상이 제대로 로드되었는지 확인

**Analytics 설정:**

- 감사 메시지에 대해 올바르게 구현된 추적 코드
- 리디렉션 대상 추적 구성됨
- 목표 완료 이벤트가 올바르게 실행됨

- **반복 및 개선**: 사용자 피드백 및 데이터 인사이트를 사용하여 사후 제출 전략을 구체화합니다
