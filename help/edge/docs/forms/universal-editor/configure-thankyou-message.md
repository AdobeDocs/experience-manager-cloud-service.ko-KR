# 감사 메시지 및 리디렉션 URL 구성

제출 후 경험은 사용자 만족도와 양식 완료율에 상당한 영향을 미칩니다. Adobe의 범용 편집기는 개인화된 감사 메시지를 통하거나 특정 페이지로의 전략적 리디렉션을 통해 사용자가 양식을 제출한 후 보는 내용을 구성하기 위한 포괄적인 옵션을 제공합니다.

이 문서는 양식 제출의 효율성을 극대화하기 위한 기술적 고려 사항, 모범 사례 및 사용자 경험 지침을 포함하여 감사 메시지와 리디렉션 URL을 모두 구현하기 위한 자세한 지침을 제공합니다.

## 사전 요구 사항

제출 후 경험을 구성하기 전에 다음 사항을 확인합니다.

**기술적 설정:**

- 적절한 권한을 가진 범용 편집기에 대한 액세스 권한
- 범용 편집기에서 만든 기존 적응형 양식
- 조직의 리디렉션 URL 요구 사항에 대한 이해

**계획 고려 사항:**

- **메시지 전략**: 감사 메시지에 포함할 톤, 길이 및 구체적인 정보를 정의합니다.
- **리디렉션 전략**: 대상 페이지를 식별하고 양식 완료 후 경험에 최적화되었는지 확인합니다.
- **분석 통합**: 감사 메시지 또는 리디렉션 대상으로의 사용자 상호 작용을 추적하는 방법을 계획합니다.

## 감사 메시지 구성

감사 메시지는 성공적인 양식 제출에 대한 즉각적인 확인을 제공하며 개인화된 콘텐츠, 다음 단계 또는 사용자의 제출과 관련된 중요한 정보를 포함할 수 있습니다.

### 감사 메시지 사용 시기

감사 메시지는 다음의 경우에 가장 효과적입니다.

- **간단한 확인**: 사용자가 추가 탐색 요구 사항 없이 확인을 필요로 하는 경우
- **교육용 콘텐츠**: 구체적인 다음 단계 또는 중요한 정보를 제공해야 하는 경우
- **브랜드 일관성**: 조직의 커뮤니케이션 스타일에 맞게 메시지를 작성할 수 있는 경우
- **단일 페이지 경험**: 워크플로 연속성을 위해 사용자가 현재 페이지에 머물러야 하는 경우

### 구현 절차

**1. 양식 속성 액세스**

범용 편집기에서 적응형 양식을 열고 도구 모음에서 **양식 속성 편집** 아이콘을 클릭합니다. 이렇게 하면 포괄적인 양식 속성 대화 상자가 열립니다.

![양식 속성 아이콘](/help/forms/assets/ue-form-properties-icon.png)

**2. 감사 구성으로 이동**

양식 속성 대화 상자에서 **감사** 탭을 선택하여 제출 후 구성 옵션에 액세스합니다.

![범용 편집기 양식 속성](/help/forms/assets/ue-form-properties.png)

**3. 메시지 표시 구성**

사용 가능한 옵션에서 **메시지 표시**&#x200B;를 선택합니다.

![thankyou](/help/edge/docs/forms/universal-editor/assets/thankyou-ue.png)

**4. 메시지 콘텐츠 만들기**

**메시지 콘텐츠** 필드에서 리치 텍스트 편집기를 사용하여 감사 메시지를 작성합니다. 편집기는 다음을 지원합니다.

- **텍스트 서식**: 굵게, 이탤릭체, 밑줄 및 색상 옵션
- **목록**: 정보를 구성하기 위한 글머리 기호 및 번호 매기기 목록
- **링크**: 관련 리소스 또는 다음 단계에 대한 직접 링크
- **전체 화면 편집**: 더 넓은 편집 작업 영역을 위해 확장 아이콘을 클릭합니다.

### 기술적 고려 사항

**메시지 표시 비헤이비어:**

- 메시지는 양식 제출 직후 모달 오버레이에 표시됩니다.
- 콘텐츠는 HTML 서식을 지원하고 반응형 디자인을 유지합니다.
- 메시지는 사용자가 해제하거나 자동 닫기 타이머로 구성할 수 있습니다.

**콘텐츠 지침:**

- 필요한 정보를 제공하면서 메시지를 간결하게 유지합니다.
- 적절한 경우 명확한 다음 단계를 포함합니다.
- 참조 번호 또는 확인 세부 정보를 포함하는 것을 고려합니다.
- 모바일 친화적인 서식인지 확인합니다.

### 구현 예시

    제출해 주셔서 감사합니다.
    
    귀하의 신청서가 접수되었으며 참조 번호 #REF-2024-001234이 지정되었습니다.
    
    **다음 단계:**
    - 15분 이내에 확인 이메일을 받게 됩니다.
    - 당사 팀에서 영업일 기준 2일 이내에 제출물을 검토합니다.
    - 추가 정보가 필요한 경우 직접 연락드립니다.
    
    **도움이 필요하십니까?** support@example.com로 지원 팀에 문의하십시오.

## 리디렉션 URL 구성

리디렉션 URL은 양식 제출 후 사용자를 특정 페이지로 자동으로 이동시켜 기존 워크플로와의 원활한 통합을 지원하거나 사용자를 관련 콘텐츠로 안내합니다.

### 리디렉션 URL 사용 시기

리디렉션 URL은 다음과 같은 경우에 적합합니다.

- **워크플로 통합**: 사용자를 대시보드, 계정 페이지 또는 프로세스의 다음 단계로 안내하는 경우
- **콘텐츠 게재**: 양식 응답을 기반으로 관련 제품, 서비스 또는 정보를 표시하는 경우
- **분석 추적**: 특정 추적 구현이 포함된 페이지로 안내하는 경우
- **다단계 프로세스**: 사용자를 복잡한 워크플로의 다음 단계로 이동시키는 경우

### 구현 절차

**1. 양식 속성 액세스**

범용 편집기에서 적응형 양식을 열고 **양식 속성 편집** 아이콘을 클릭하여 양식 구성 대화 상자를 엽니다.

![양식 속성 아이콘](/help/forms/assets/ue-form-properties-icon.png)

**2. 감사 구성으로 이동**

양식 속성 대화 상자에서 **감사** 탭을 선택하여 리디렉션 구성 옵션에 액세스합니다.

![범용 편집기 양식 속성](/help/forms/assets/ue-form-properties.png)

**3. 리디렉션 기능 활성화**

사용 가능한 제출 후 옵션에서 **URL로 리디렉션**&#x200B;을 선택합니다.

![redirect](/help/edge/docs/forms/universal-editor/assets/redirect-ue.png)

**4. 대상 URL 구성**

제공된 필드에 대상 URL을 입력합니다. 시스템은 유연한 구현을 위해 여러 URL 형식을 지원합니다.

### URL 구성 옵션

**절대 URL**

프로토콜 및 도메인을 포함한 전체 웹 주소:

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**상대 경로**

현재 도메인에 대한 상대 경로:

    /thank-you
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sites 페이지 참조**

AEM Sites 구현 내 다른 페이지에 대한 참조:

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 기술적 고려 사항

**리디렉션 비헤이비어:**

- 리디렉션은 양식 제출 성공 직후 발생합니다.
- 브라우저 기록에 리디렉션이 포함되어 뒤로 가기 버튼 기능이 제대로 작동합니다.
- 리디렉션 타이밍은 선택적 지연으로 구성할 수 있습니다.

**URL 유효성 검사:**

- 시스템은 구성을 허용하기 전에 URL 형식을 유효성 검사합니다.
- 상대 URL은 현재 도메인에 대해 확인됩니다.
- 외부 URL에는 경우에 따라 적절한 CORS 구성이 필요합니다.

## 모범 사례 및 권장 사항

### 사용자 경험 지침

**메시지 최적화:**

- **명확성 우선**: 사용자가 제출이 성공적으로 완료되었음을 즉시 이해하도록 합니다.
- **가치 추가**: 사용자가 다음 단계를 수행하는 데 도움이 되는 정보를 제공합니다.
- **일관된 브랜딩**: 조직의 음성 및 시각적 스타일을 유지합니다.
- **모바일 고려**: 다양한 화면 크기에서 메시지를 테스트합니다.

**리디렉션 최적화:**

- **페이지 최적화**: 리디렉션 대상이 양식 제출 후 방문자에게 최적화되었는지 확인합니다.
- **로딩 성능**: 사용자 경험을 유지하도록 리디렉션 페이지가 빠르게 로드되는지 확인합니다.
- **콘텐츠 관련성**: 리디렉션 콘텐츠가 양식 컨텍스트와 관련이 있는지 확인합니다.

### 보안 고려 사항

**URL 유효성 검사:**

- 악성 리디렉션을 방지하기 위해 리디렉션 URL에 대한 적절한 유효성 검사를 구현합니다.
- 허용된 리디렉션 도메인에 대해 허용 목록 방식을 고려합니다.
- 비정상적인 활동에 대한 리디렉션 패턴을 모니터링합니다.

**콘텐츠 보안:**

- 스크립트 삽입을 방지하기 위해 감사 메시지 콘텐츠를 정리합니다.
- 리치 텍스트 콘텐츠에 대한 적절한 콘텐츠 보안 정책을 구현합니다.
- 리디렉션 대상에 대한 정기적인 보안 검토를 수행합니다.

### 분석 및 추적

**구현 고려 사항:**

- **목표 추적**: 감사 메시지 보기 및 리디렉션 완료에 대한 분석 목표를 설정합니다.
- **사용자 여정 매핑**: 제출 후 경험과 사용자가 상호 작용하는 방식을 추적합니다.
- **전환 최적화**: 다양한 감사 메시지 및 리디렉션 대상을 A/B 테스트합니다.

**측정 전략:**

- 해제하기 전 감사 메시지에 소요된 시간을 모니터링합니다.
- 감사 메시지 내 링크에 대한 클릭스루 비율을 추적합니다.
- 리디렉션 대상 페이지에서 사용자 비헤이비어를 분석합니다.

## 유효성 검사 체크포인트

제출 후 환경을 구성한 후 다음 작업을 수행합니다.

**구성 확인:**

- 양식 속성에서 선택한 감사 옵션이 올바르게 표시됩니다.
- 미리보기 모드에서 메시지 콘텐츠가 올바르게 표시됩니다.
- 리디렉션 URL의 형식이 제대로 지정되었으며 액세스 가능합니다.
- 메시지 내의 모든 링크가 올바르게 작동합니다.

**사용자 경험 테스트:**

- 테스트 양식을 제출하여 적절한 감사 메시지 표시 여부를 확인합니다.
- 다양한 브라우저에서 리디렉션 기능을 테스트합니다.
- 감사 메시지의 모바일 반응성을 확인합니다.
- 리디렉션 대상이 올바르게 로드되는지 확인합니다.

**Analytics 설정:**

- 감사 메시지에 대해 추적 코드가 올바르게 구현되었습니다.
- 리디렉션 대상 추적이 구성되었습니다.
- 목표 완료 이벤트가 올바르게 실행됩니다.

## 다음 단계

제출 후 환경을 성공적으로 구성한 후 다음 작업을 수행합니다.

- **성능 모니터링**: 분석을 검토하여 감사 메시지 또는 리디렉션 페이지에 대한 사용자 참여를 파악합니다.
- **반복 및 개선**: 사용자 피드백 및 데이터 인사이트를 사용하여 제출 후 전략을 개선합니다.
- **구현 확장**: 조직의 다른 양식에 성공적인 패턴을 적용합니다.
