# Forms Experience Builder - 문제 해결 안내서

>[!NOTE]
>
> Forms Experience Builder는 얼리 어답터 프로그램에서 사용할 수 있습니다. 액세스 권한을 요청하려면 회사 이메일 주소를 통해 `aem-forms-ea@adobe.com`으로 이메일을 보내시기 바랍니다.

>[!IMPORTANT]
>
> **변경될 수 있는 설명서**: 이 문제 해결 안내서는 현재 제품과의 일치 여부를 테스트 중이며 업데이트 및 개정이 이루어질 수 있습니다. 얼리 어답터 프로그램 동안 Forms Experience Builder가 계속 발전함에 따라 문제, 솔루션 및 디버깅 기술이 변경될 수 있습니다.

이 포괄적인 문제 해결 안내서는 Forms Experience Builder를 사용할 때 흔히 발생하는 문제를 식별, 진단, 해결하는 데 도움을 줍니다. 이 안내서는 문제 카테고리별로 구성되어 있으며, 빠른 해결책과 자세한 솔루션을 함께 제공합니다.

## 빠른 참조 - 일반적인 문제

| 문제 | 빠른 해결책 |
|-------|-----------|
| **인터페이스가 로드되지 않음** | 브라우저 새로 고침, 인터넷 연결 확인, 얼리 액세스 권한 확인 |
| **명령이 작동하지 않음** | `/help` 시도 또는 슬래시 명령 대신 자연어 사용 |
| **@fieldName을 인식할 수 없음** | 철자 확인, 필드가 존재하는지 먼저 확인, 필드 이름 구문 확인 |
| **파일 업로드 실패** | 10MB 이하의 PDF/JPG/PNG 사용, 파일 포맷 호환성 확인 |
| **양식이 잘못됨** | 더 구체적으로 설명: “레이아웃 수정” 대신 “모바일 친화적으로 만들기” |
| **통합 실패** | API 자격 증명 및 권한 확인, 엔드포인트 가용성 확인 |
| **양식이 제출되지 않음** | 제출 액션 구성 및 유효성 검사 규칙 확인 |
| **유효성 검사 오류가 표시되지 않음** | 필드 유효성 검사 설정 및 오류 메시지 배치 확인 |
| **모바일 레이아웃 문제** | 반응형 디자인 설정 및 필드 크기 검토 |
| **필드가 나타나지 않음** | 조건 논리 및 가시성 규칙 확인 |
| **가져오기 실패** | 파일 포맷 호환성 및 크기 제한 확인 |
| **성능 문제** | 필드 수 최적화 및 불필요한 유효성 검사 제거 |
| **접근성 문제** | 필드 레이블, ARIA 속성 및 탭 순서 검토 |

**여전히 도움이 필요하신가요?** `/help` 다음에 구체적인 질문을 입력하거나 시스템 관리자에게 기술 지원을 요청합니다.

## 액세스 및 인증 문제

### Forms Experience Builder에 액세스할 수 없음

**증상:**

- Forms Experience Builder 인터페이스가 보이지 않음
- “액세스 거부됨” 또는 이와 유사한 오류 메시지
- 편집기에서 Forms Experience Builder 아이콘 누락

**솔루션:**

1. **얼리 액세스 프로그램 등록 확인**
   - 얼리 어답터 프로그램에 승인되었는지 확인
   - 요청이 공식 업무 이메일에서 전송되었는지 확인
   - 액세스가 아직 보류 중인 경우 `aem-forms-ea@adobe.com`으로 문의

2. **환경 설정 확인**
   - 사용자 환경에서 AEM Forms가 활성화되어 있는지 확인
   - 지원되는 브라우저(Chrome, Firefox, Safari, Edge)를 사용하고 있는지 확인
   - 브라우저 캐시 및 쿠키 지우기
   - 방해가 될 수 있는 브라우저 확장 기능 비활성화

3. **사용자 권한 확인**
   - 적절한 사용자 역할 및 권한이 있는지 확인
   - 시스템 관리자에게 액세스 권한에 대해 문의
   - 올바른 계정으로 로그인했는지 확인

### 인터페이스 로딩 문제

**증상:**

- 빈 인터페이스 또는 부분적으로 로드된 인터페이스
- 완료되지 않는 회전 로드 표시기
- 브라우저 콘솔의 JavaScript 오류

**솔루션:**

1. **브라우저 문제 해결**
   - 페이지 새로 고침 (Ctrl+F5 또는 Cmd+Shift+R)
   - 다른 브라우저나 시크릿/비공개 모드 사용해 보기
   - 브라우저 업데이트 확인 및 가능하면 설치
   - 일시적으로 광고 차단기 및 개인 정보 보호 확장 기능 비활성화

2. **네트워크 연결**
   - 안정적인 인터넷 연결 확인
   - 회사 방화벽이 필수 도메인을 차단하고 있는지 확인
   - 가능하다면 다른 네트워크 연결로 테스트
   - 네트워크 구성 문제는 IT 지원팀에 문의

3. **캐시 및 스토리지 문제**
   - 브라우저 캐시 및 로컬 저장소 지우기
   - 브라우저 설정을 기본값으로 재설정
   - 디바이스에서 사용 가능한 디스크 공간 확인
   - 다양한 디바이스에서 액세스해 보기

## 명령 및 상호 작용 문제

### 슬래시 명령이 작동하지 않음

**증상:**

- `/create-form` 또는 기타 슬래시 명령을 인지하지 못함
- 자동 완성 제안이 나타나지 않음
- 명령어로 인해 오류 메시지 표시

**솔루션:**

1. **명령어 구문 확인**
   - 적절한 명령 포맷 확인: `/command-name description`
   - 명령어 이름에 오타가 있는지 확인
   - 대안으로 자연어 사용: “연락처 양식 만들기”
   - `/help`로 명령 가용성 확인

2. **컨텍스트에 맞는 명령**
   - 올바른 편집기 컨텍스트에 있는지 확인 (범용 편집기와 Adaptive Forms Editor)
   - 일부 명령의 경우 특정 환경에서만 작동
   - 컨텍스트 요구 사항에 대한 명령 참조 확인

3. **대안적 접근 방식**
   - 슬래시 명령 대신 자연어 사용
   - 복잡한 명령을 더 작고 간단한 요청으로 분할
   - 복잡한 단일 명령 대신 단계별 양식 작성 시도

### 필드 참조가 작동하지 않음

**증상:**

- `@fieldName` 참조가 인식되지 않음
- 알 수 없는 필드에 대한 오류 메시지
- 필드 수정 사항이 올바르게 적용되지 않음

**솔루션:**

1. **필드 이름 확인**
   - 필드 이름의 정확한 철자 확인 (대소문자 구분)
   - 참조하기 전에 필드가 존재하는지 확인
   - 표시 레이블이 아닌 생성된 정확한 필드 이름 사용
   - 필드 명명 규칙(camelCase, snake_case 등) 확인

2. **필드 참조 구문**
   - 공백 없이 적절한 `@fieldName` 구문 사용
   - 필드 참조에 특수 문자 피하기
   - 보이지 않는 문자나 서식 문제가 있는지 확인
   - 필드 참조를 수동으로 시도

3. **디버깅 필드 참조**
   - 기존 필드를 모두 먼저 나열: “현재 양식 필드를 모두 표시”
   - 규칙에서 참조하기 전 필드 만들기
   - 복잡한 문자 없이 간단한 필드 이름 사용
   - 한 번에 하나씩 필드 참조 테스트

## 양식 생성 및 디자인 문제

### 예상대로 양식이 생성되지 않음

**증상:**

- 요청된 필드가 없는 생성된 양식
- 잘못된 필드 유형 또는 레이아웃
- 양식 구조가 설명과 일치하지 않음

**솔루션:**

1. **프롬프트 구체성 개선**
   - 양식 설명을 더 자세히 작성
   - 정확한 필드 유형 및 유효성 검사 요구 사항 지정
   - 레이아웃 기본 설정 및 사용자 경험 요구 사항 포함
   - 복잡한 양식을 더 작고 점진적인 요청으로 분할

2. **반복적 개발 접근 방식**
   - 기본 양식 구조로 시작
   - 필드와 기능을 점진적으로 추가
   - 진행하기 전에 각 추가 사항 테스트
   - 단일 복잡한 요청보다는 대화를 통해 구체화

3. **더 나은 프롬프트의 예**

   다음 대신에:

       고객을 위한 양식 만들기
   
   사용:

       다음을 사용하여 고객 문의 양식 만들기:
       - 성명 (필수 텍스트 필드)
       - 이메일 주소 (검증 시 필수)
       - 전화번호 (선택 사항, 형식 지정)
       - 메시지 (필수 텍스트 영역, 최대 500자)
       - 이메일 알림으로 제출
   
### 레이아웃 및 스타일링 문제

**증상:**

- 양식이 모바일 디바이스에서 잘려 보임
- 일관되지 않은 간격 또는 정렬
- 필드가 정확하게 표시되지 않음
- 시각적 계층이 좋지 않음

**솔루션:**

1. **모바일 반응성**
   - 모바일에 적용 최적화 요청: “이 양식을 모바일 친화적으로 만들기”
   - 반응형 디자인 요구 사항 지정
   - 실제 모바일 디바이스에서 테스트
   - 모바일에 단일 열 레이아웃 사용

2. **레이아웃 개선**
   - 레이아웃 요구 사항에 대해 구체적으로 설명: “주소 필드를 두 열로 정렬”
   - 구체적으로 스타일링 요청: “전문적인 색상과 간결한 타이포그래피 사용”
   - 간격 및 정렬 요구 사항 지정
   - 접근성 규정 준수 요청

3. **브랜드 일관성**
   - 양식 생성 전 브랜드 가이드라인 준비
   - 요청에 구체적인 색상 코드와 글꼴 포함
   - 모든 양식에서 일관된 스타일링 사용
   - 재사용을 위한 브랜드 템플릿 만들기

### 조건 논리 문제

**증상:**

- 예상대로 규칙이 트리거되지 않음
- 필드가 잘못 표시되거나 숨겨짐
- 유효성 검사 논리가 작동하지 않음
- 복잡한 비즈니스 규칙 실패

**솔루션:**

1. **규칙 단순화**
   - 복잡한 규칙을 더 작고 간단한 조건으로 분해
   - 결합 전에 각 규칙을 개별적으로 테스트
   - 명확하고 구체적인 조건 사용: “@maritalStatus가 “기혼”일 때 @spouseInfo 표시”
   - 처음에는 중첩되거나 지나치게 복잡한 논리 피하기

2. **규칙 테스트 및 디버깅**
   - 가능한 모든 사용자 경로와 시나리오 테스트
   - 조건에서 필드 이름과 값 확인
   - 규칙 조건에서 대소문자 구분 확인
   - 디버그 모드를 사용하여 규칙 실행 추적

3. **비즈니스 논리 구현**
   - 구현 전에 비즈니스 요구 사항을 명확하게 문서화
   - 규칙을 점진적으로 구현하고 각 단계를 테스트
   - 규칙이 트리거되면 명확한 사용자 피드백 제공
   - 극단적 사례 및 예외 시나리오 처리

## 파일 가져오기 및 변환 문제

### PDF 가져오기 실패

**증상:**

- PDF 파일이 업로드되지 않거나 처리되지 않음
- 변환된 양식에 필드 또는 콘텐츠가 누락됨
- PDF 변환 중 오류 메시지
- PDF에서 필드 인식 저하

**솔루션:**

1. **파일 포맷 및 크기**
   - PDF 파일이 10MB 이하인지 확인
   - 고품질의 텍스트 기반 PDF 사용 (스캔한 이미지 아님)
   - PDF가 암호로 보호되거나 암호화되지 않았는지 확인
   - 텍스트 추출이 실패하면 PDF를 이미지 포맷으로 변환해 보기

2. **PDF 품질 최적화**
   - 명확하고 잘 정의된 양식 필드가 있는 PDF 사용
   - 좋은 대비와 읽기 쉬운 텍스트 확보
   - 복잡한 레이아웃이나 겹치는 요소 피하기
   - 전환 요청에 추가 컨텍스트 제공

3. **전환 향상**
   - 예상되는 양식 구조를 상세히 설명
   - 필드 유형 및 유효성 검사 요구 사항 지정
   - 구체적인 개선 사항 요청: “모바일 반응성 및 유효성 검사 추가”
   - 변환된 양식을 수동으로 검토 및 수정

### 이미지 및 스크린샷 변환 문제

**증상:**

- 이미지에서 필드 인식 저하
- 잘못된 필드 유형 또는 레이아웃
- 양식 요소 누락
- 변환 오류 또는 시간 초과

**솔루션:**

1. **이미지 품질 요구 사항**
   - 고해상도 이미지(최소 300 DPI) 사용
   - 좋은 조명과 대비 확보
   - 그림자, 눈부심 또는 왜곡 피하기
   - 양식 콘텐츠에만 초점을 맞추기 위해 이미지 자르기

2. **최적의 이미지 포맷**
   - 최상의 결과를 얻으려면 PNG 또는 JPG 포맷 사용
   - GIF 또는 품질이 낮은 압축 이미지 피하기
   - 이미지에서 텍스트를 명확하게 읽을 수 있는지 확인
   - 필요한 경우 다른 이미지 방향 시도

3. **전환 지침**
   - 양식 구조에 대한 자세한 설명 제공
   - 필드 유형과 요구 사항을 명시적으로 지정
   - 변환 중 특정 개선 사항 요청
   - 변환 후 수동 조정 준비

## 통합 및 제출 문제

### 양식 제출 실패

**증상:**

- 양식이 성공적으로 제출되지 않음
- 제출 중 오류 메시지
- 의도한 목적지에 데이터가 도달하지 않음
- 제출 중 시간 초과 오류

**솔루션:**

1. **제출 - 액션 구성**
   - 제출 액션이 올바르게 구성되었는지 확인
   - API 엔드포인트와 인증 자격 증명 확인
   - 먼저 간단한 이메일 제출로 테스트
   - 데이터 포맷 요구 사항 확인

2. **네트워크 및 연결**
   - 인터넷 연결 및 네트워크 안정성 확인
   - 방화벽 설정이 양식 제출을 허용하는지 확인
   - 다양한 네트워크 연결에서 테스트
   - 기업 프록시 또는 보안 제한 사항 확인

3. **데이터 유효성 검사 문제**
   - 모든 필수 필드가 완료되었는지 확인
   - 데이터 포맷이 API 요구 사항과 일치하는지 확인
   - 특수 문자 또는 인코딩 문제 확인
   - 먼저 최소한의 데이터 세트로 테스트

### API 통합 문제

**증상:**

- REST API 엔드포인트가 응답하지 않음
- 인증 실패
- 데이터 포맷 불일치
- 통합 시간 초과 또는 오류

**솔루션:**

1. **API 구성 확인**
   - API 엔드포인트 URL이 올바르고 액세스 가능한지 확인
   - 인증 자격 증명 및 권한 확인
   - Postman과 같은 도구를 사용하여 API 엔드포인트를 독립적으로 테스트
   - API가 올바른 데이터 포맷(JSON, XML 등)을 수용하는지 확인

2. **데이터 매핑 문제**
   - 양식 필드 이름이 API 매개변수 요구 사항과 일치하는지 확인
   - 필수 필드가 누락되었는지 확인
   - 데이터 유형 호환성(문자열, 숫자, 날짜) 확인
   - 매핑 문제를 식별하기 위해 샘플 데이터로 테스트

3. **오류 처리 및 디버깅**
   - API 호출에 대한 상세한 오류 로깅 확인
   - API 응답 코드와 오류 메시지 확인
   - 일시적인 실패에 대한 재시도 논리 구현
   - API가 실패할 경우 사용자에게 대체 옵션 제공

### 이메일 통합 문제

**증상:**

- 확인 이메일이 전송되지 않음
- 이메일이 스팸 폴더로 이동
- 이메일 형식이 올바르지 않음
- 이메일에 양식 데이터가 누락됨

**솔루션:**

1. **이메일 구성**
   - 이메일 주소가 올바른 형식인지 확인
   - SMTP 설정 및 인증 확인
   - 먼저 간단한 이메일 주소로 테스트
   - 이메일 서버 권한 및 할당량 확인

2. **이메일 게재 최적화**
   - 적절한 이메일 헤더와 발신자 정보 사용
   - 제목줄에 스팸 트리거 단어 피하기
   - 적절한 구독 취소 메커니즘 포함
   - 다양한 공급자에 대한 이메일 게재 테스트

3. **콘텐츠 및 서식**
   - 이메일의 양식 데이터가 올바르게 서식 지정되었는지 확인
   - 특수 문자 또는 인코딩 문제 확인
   - 다양한 데이터 조합으로 이메일 템플릿 테스트
   - 이메일 콘텐츠가 액세스 가능하고 읽기 쉬운지 확인

## 성능 및 로딩 문제

### 느린 양식 로딩

**증상:**

- 양식을 처음 로드하는 데 시간이 오래 걸림
- 느린 사용자 상호 작용
- 양식 작업 중 시간 초과
- 모바일 디바이스에서의 성능 저하

**솔루션:**

1. **양식 최적화**
   - 필드 수와 복잡성 줄이기
   - 비중요 섹션에 대한 지연 로딩 구현
   - 웹 게재를 위한 이미지와 자산 최적화
   - 불필요한 유효성 검사 규칙이나 논리 제거

2. **브라우저 및 디바이스 최적화**
   - 브라우저 캐시 및 임시 파일 지우기
   - 불필요한 브라우저 탭과 애플리케이션 닫기
   - 사용 가능한 디바이스 메모리 및 저장 용량 확인
   - 다양한 브라우저를 사용해 성능 비교

3. **네트워크 최적화**
   - 다양한 네트워크 연결로 테스트
   - 네트워크 혼잡이나 대역폭 제한 확인
   - 가능하면 WiFi 대신 유선 연결 사용
   - 네트워크 성능 문제는 IT 지원팀에 문의

### 유효성 검사 성능 문제

**증상:**

- 느린 유효성 검사 응답
- 오류 메시지 지연 표시
- 유효성 검사 중 양식 중단
- 필드 유효성 검사 중 시간 초과 오류

**솔루션:**

1. **유효성 검사 최적화**
   - 실시간 유효성 검사 빈도 줄이기
   - 유효성 검사 호출에 대한 디바운싱 구현
   - 복잡한 유효성 검사 규칙 단순화
   - 가능한 경우 클라이언트측 유효성 검사 사용

2. **규칙 단순화**
   - 복잡한 검증을 더 작은 규칙으로 분할
   - 불필요한 교차 필드 유효성 검사 제거
   - 성능을 위한 조건 논리 최적화
   - 적절한 경우 캐시 유효성 검사 결과

3. **사용자 경험 개선**
   - 간단한 유효성 검사에 대한 즉각적인 피드백 제공
   - 복잡한 규칙에 대해 실시간 대신 점진적 검증 사용
   - 유효성 검사 프로세스 중 로딩 표시기 표시
   - 사용자가 백그라운드에서 유효성 검사 프로세스를 계속할 수 있도록 허용

## 고급 문제 해결

### 디버그 모드 및 진단

**디버그 정보 활성화**

다음 프롬프트를 사용하여 양식 문제에 대한 자세한 정보 얻기:

    양식 제출 및 필드 유효성 검사로 문제를 식별하려면 디버그 모드를 활성화
    
    양식 오류 분석: 유효성 검사 규칙, API 응답 및 사용자 입력 패턴 확인
    
    양식 구조 및 필드 구성에 대한 자세한 정보 표시

### 오류 분석 기술

**체계적인 디버깅 접근 방식**

1. **문제 분리**
   - 최소한의 양식 구성으로 테스트
   - 복잡한 기능 일시적으로 제거
   - 개별 구성 요소 별도로 테스트
   - 제거 프로세스를 사용하여 근본 원인 확인

2. **진단 정보 수집**
   - 브라우저 콘솔에서 JavaScript 오류 확인
   - 네트워크 요청 및 응답 검토
   - 문제를 재현하기 위한 정확한 단계 문서화
   - 스크린샷과 오류 메시지 수집

3. **환경 변수 테스트**
   - 다양한 브라우저와 디바이스에서 사용해 보기
   - 다양한 사용자 계정 및 권한으로 테스트
   - 다양한 네트워크 환경에서 확인
   - 작업 양식 또는 구성과 비교

### 로그 분석 및 모니터링

**브라우저 콘솔 디버깅**

1. 브라우저 개발자 도구(F12) 열기
2. 콘솔 탭에서 JavaScript 오류 확인
3. 네트워크 탭에서 실패한 요청 검토
4. 성능 탭에서 느린 작업 모니터링

**일반적인 오류 패턴**

- **CORS 오류**: API 통합과 관련된 교차 출처 요청 문제
- **인증 실패**: 잘못된 자격 증명 또는 만료된 토큰
- **유효성 검사 오류**: 필드 유효성 검사 규칙 충돌 또는 구문 오류
- **네트워크 시간 초과**: 느리거나 신뢰할 수 없는 네트워크 연결
