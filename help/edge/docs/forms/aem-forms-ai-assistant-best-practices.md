# Forms Experience Builder - 모범 사례

>[!NOTE]
>
> Forms Experience Builder는 얼리 어답터 프로그램에서 사용할 수 있습니다. 액세스 권한을 요청하려면 회사 이메일 주소를 통해 `aem-forms-ea@adobe.com`으로 이메일을 보내시기 바랍니다.

>[!IMPORTANT]
>
> **변경될 수 있는 설명서**: 이 모범 사례 안내서는 현재 제품과의 일치 여부를 테스트 중이며 업데이트 및 개정이 이루어질 수 있습니다. 얼리 어답터 프로그램이 진행되는 동안 Forms Experience Builder가 계속 발전함에 따라 모범 사례, 권장 사항 및 예시가 변경될 수 있습니다.

이 종합 안내서는 Forms Experience Builder를 사용하여 효과적이고 사용자 친화적인 양식을 만드는 데 대한 검증된 모범 사례를 제공합니다. 이러한 모범 사례는 다양한 산업 및 사용 사례에서의 성공적인 구현과 사용자 피드백을 통해 도출됩니다.

## 양식 디자인 모범 사례

### 단순하게 유지

**필수 분야부터 시작**

- 사용자의 과부하를 줄이기 위해 가장 필요한 정보로만 시작
- 사용자 요구 사항 및 비즈니스 요구 사항에 따라 점진적으로 복잡성 추가
- 실제로 필요하지 않거나 사용하지 않는 정보 묻지 않기
- 양식을 디자인할 때 사용자의 시간과 인지 부하 고려

**명확하고 설명적인 레이블 사용**

- 설명적인 레이블을 사용하여 필드 목적을 즉시 명확하게 표시
- 사용자가 이해하지 못하는 기술 용어나 내부 용어 피하기
- 양식 전체에 일관된 레이블 지정 규칙 사용
- 필드 요구 사항이 명확하지 않을 경우 컨텍스트 제공

**도움이 되는 지침 제공**

- 복잡한 필드에 대한 도움말 텍스트를 예시와 서식 요구 사항과 함께 포함
- 플레이스홀더 텍스트를 사용하여 예상 입력 포맷 표시
- 허용되는 포맷과 크기 제한 등 파일 업로드에 대한 명확한 지침 제공
- 진행률 표시기를 사용하여 여러 단계의 프로세스 안내

**철저히 테스트**

- 배포 전 모든 사용자 경로와 시나리오 검증
- 타깃 대상자의 실제 사용자를 대상으로 양식 테스트
- 모든 조건 논리가 예상대로 작동하는지 확인
- 오류 처리가 명확하고 실행 가능한 피드백을 제공하는지 확인

### 점진적 공개

**컨텍스트에 따라 관련 필드 표시**

- 사용자 선택과 관련이 있을 때만 추가 필드 표시
- 조건 논리를 사용하여 인지 부하 및 양식 길이 줄이기
- 논리적 섹션에 관련 필드 그룹화
- 사용자가 필요하다고 표시할 때까지 고급 옵션 숨기기

**구현 예시:**

    @maritalStatus가 “기혼”일 때만 @spouseInformation 패널을 표시하는 규칙 만들기

**명확한 탐색 및 진행**

- 사용자가 여러 단계로 구성된 양식에서 어디에 있는지 이해하도록 돕기
- 양식 섹션 간에 명확한 탐색 제공
- 긴 양식에 대해 진행률 표시기 표시
- 사용자가 진행 상황을 저장하고 나중에 다시 돌아올 수 있도록 허용

## 사용자 경험 모범 사례

### 모바일 우선 디자인

**반응형 레이아웃 최적화**

- 모바일 사용자를 주요 고려 사항으로 하여 양식 디자인
- 모바일 디바이스에서 단일 열 레이아웃 사용
- 터치 대상의 크기가 적절한지 확인 (최소 44px)
- 다양한 디바이스 크기 및 방향에 대해 양식 테스트

**터치 친화적인 인터랙션**

- 터치 인터페이스에 맞는 더 큰 버튼과 입력 필드 구현
- 적절한 입력 유형을 사용하여 올바른 모바일 키보드 트리거
- 터치 디바이스에서 작동하지 않는 호버 의존 인터랙션 피하기
- 사용자 인터랙션에 대한 명확한 시각적 피드백 제공

### 접근성 규정 준수

**WCAG 2.1 지침**

- 포괄적 디자인을 위한 웹 콘텐츠 접근성 지침 따르기
- 적절한 색상 대비율 확보 (일반 텍스트의 경우 최소 4.5:1)
- 모든 이미지와 아이콘에 대한 그림 설명 제공
- 적절한 제목 구조와 유의미한 HTML 구현

**키보드 탐색**

- 키보드 탐색을 통해 모든 양식 요소에 액세스할 수 있는지 확인
- 모든 대화형 요소에 대해 명확한 초점 표시기 제공
- 양식 필드를 통한 논리적 탭 순서 구현
- 복잡한 양식에 탐색 건너뛰기 링크 포함

**화면 판독기 지원**

- 양식 필드에 적절한 ARIA 레이블과 설명 사용
- 화면 판독기에 전달되는 명확한 오류 메시지 제공
- 동적 콘텐츠 변경 사항이 적절하게 발표되었는지 확인
- 실제 화면 판독기 소프트웨어를 사용해 양식 테스트

### 성능 최적화

**로딩 속도**

- 초기 번들 크기를 최소화하여 양식 로딩 시간 최적화
- 비중요 양식 섹션에 대한 지연 로딩 구현
- 웹 게재를 위한 이미지와 자산 최적화
- 정적 리소스에 대한 적절한 캐싱 전략 활성화

**런타임 성능**

- 적절한 유효성 검사 타이밍을 사용하여 사용자 경험과 성능 균형
- 서버 요청을 줄이기 위해 실시간 유효성 검사에 대한 디바운싱 구현
- 자주 액세스되는 데이터 및 유효성 검사 결과 캐시
- 복잡한 양식에 대한 조건 논리 실행 최적화

**자동 저장 및 데이터 보호**

- 데이터 손실을 방지하기 위해 양식 진행 상황을 자동으로 저장하도록 구현
- 가능한 경우 양식 작성을 위해 오프라인 기능 제공
- 실패한 제출에 대해 재시도 논리 포함
- 사용자를 위한 백업으로 데이터 내보내기 옵션 제공

## 브랜드 일관성 모범 사례

### 브랜드 자산 미리 준비

**브랜드 템플릿 만들기**

- 조직의 시각적 정체성을 반영한 표준화된 양식 템플릿 개발
- 일관된 색상 구성표, 타이포그래피 및 레이아웃 패턴 포함
- 브랜드 가이드라인에 맞는 재사용 가능한 구성 요소 준비
- 팀 간 일관된 구현을 위한 문서 스타일 표준

**스타일 가이드라인 정의**

- 일관된 필드 스타일, 버튼 디자인 및 간격 표준 정립
- 색상 코드, 글꼴 사양 및 레이아웃 규칙을 포함하는 스타일 안내서 만들기
- 인터랙션 패턴과 애니메이션 가이드라인 정의
- 브랜드별 오류 메시지 및 도움말 텍스트 준비

**구성 요소 라이브러리 빌드**

- 브랜드 아이덴티티와 일치하는 재사용 가능한 양식 구성 요소 만들기
- 일관된 탐색 패턴과 사용자 인터페이스 요소 개발
- 양식 통합을 위해 브랜드 아이콘, 로고 및 시각적 자산 준비
- 일반적인 양식 유형(연락처, 등록, 피드백)에 대한 템플릿 설정

### 브랜드 구현 전략

**일관성을 위한 스타일 프롬프트**

프롬프트에 브랜드별 지침 포함

    다음을 사용하여 전문적인 연락처 양식 만들기:
    - 기본 버튼 및 헤더에 Corporate Blue (#003366)
    - 모든 텍스트 요소에 Open Sans 글꼴 패밀리
    - 접근성 규정 준수를 위해 최소 16px 글꼴 크기
    - 양식 섹션 간 일관된 24px 간격
    - 적절한 브랜드 위치가 지정된 헤더의 회사 로고

**템플릿 라이브러리 관리**

- 일반적인 사용 사례에 대해 브랜드화된 양식 템플릿 컬렉션 유지 관리
- 일관성을 보장하기 위해 브랜드 템플릿의 버전 제어
- 템플릿 사용 및 사용자 정의에 대해 명확한 문서 제공
- 현재 표준을 유지하기 위해 브랜드 자산을 정기적으로 검토하고 업데이트

>
>**사용자 정의 구성 요소**: 사용자 정의 브랜드 요소를 구현하기 전에 조직별 구성 요소 사용 및 Forms Experience Builder와의 호환성에 대해 개발 팀과 협의하십시오.

## 콘텐츠 및 커뮤니케이션 모범 사례

### 양식 생성 접근 방식

**두 가지 주요 방법**

요구 사항에 가장 적합한 생성 방법 선택:

1. **처음부터 만들기**: 특정 요구 사항이 있는 새 양식에 가장 적합
2. **가져오기 및 변환**: 기존 양식 및 문서를 현대화하는 데 이상적

**자연어 프롬프트**

- 양식 설명에는 구체적이고 자세하게 작성
- 기술 용어보다는 명확하고 비즈니스 중심적인 언어 사용
- 양식의 목적과 타깃 대상자에 대한 컨텍스트 제공
- 초기 프롬프트에 유효성 검사 및 비즈니스 규칙 요구 사항 포함

### 점진적 개발 전략

**간단하게 시작해서 복잡하게 확장**

- 기본 양식 구조와 필수 필드로 시작
- 유효성 검사 규칙과 비즈니스 로직을 점진적으로 추가
- 다음 향상 작업으로 진행하기 전에 각 추가 사항 테스트
- 개발의 각 단계에서 사용자 피드백 수집

**점진적 접근 방식의 예:**

    1단계: “이름, 이메일, 메시지 필드가 있는 기본 연락처 양식 만들기”
    2단계: “적절한 검증을 통해 @name 및 @email을 필수 필드로 만들기”
    3단계: “사용자 안내를 위한 플레이스홀더 텍스트와 도움말 텍스트 추가”
    4단계: “문의 유형에 따라 조건 논리 추가”

### 필드 참조 모범 사례

**일관된 명명 규칙**

- 목적에 맞는 명확하고 설명적인 필드 이름 사용
- 양식 전체에서 일관된 명명 패턴 유지
- 양식 전체에서 camelCase 또는 snake_case를 일관되게 사용
- 팀 일관성을 위한 문서 필드 명명 규칙

**효과적인 필드 참조**

- 기존 필드를 수정할 때 `@fieldName` 구문 사용
- 복잡한 양식에서 참조하는 필드를 구체적으로 지정
- 단일 요청의 관련 필드 수정 사항을 그룹화
- 조건 논리에서 필드 이름을 사용하기 전 필드 이름 확인

## 통합 및 제출 모범 사례

### 멀티채널 제출 전략

**기본 및 보조 액션**

- 핵심 비즈니스 프로세스에 대한 기본 제출 엔드포인트 구성
- 알림, 확인 및 데이터 백업을 위한 보조 액션 설정
- 실패한 제출에 대한 오류 처리 및 재시도 논리 구현
- 모든 제출 상태(성공, 실패, 처리)에 대한 사용자 피드백 제공

**통합 계획**

- 기본 제출 구성으로 시작하여 점진적으로 통합 추가
- 여러 엔드포인트를 결합하기 전에 각 통합을 별도로 테스트
- 문서 API 요구 사항 및 인증 방법
- 데이터 변환 및 매핑 요구 사항 계획

### 오류 처리 및 복구

**알기 쉬운 오류 메시지.**

- 사용자가 문제를 해결하는 데 도움이 되는 명확하고 실행 가능한 오류 메시지 제공
- 사용자 중심 콘텐츠에 기술적 오류 코드나 시스템 메시지 표시하지 않기
- 기본 제출이 실패할 경우 대체 액션 제공
- 추가 도움이 필요한 사용자를 위한 연락처 정보 포함

**데이터 보호 및 복구**

- 오류 발생 후 양식 복구를 위한 로컬 데이터 저장 구현
- 사용자가 양식 데이터를 백업으로 다운로드할 수 있는 옵션 제공
- 제출 실패에 대한 모니터링 및 경고 설정
- 시스템 중단 또는 유지 관리에 대한 복구 절차 계획

## 성능 및 분석 모범 사례

### 모니터링 및 최적화

**주요 성과 지표**

- 양식 완료율과 포기 지점 추적
- 로딩 시간과 사용자 인터랙션 패턴 모니터링
- 유효성 검사 효과 및 오류율 측정
- 사용자 피드백 및 만족도 점수 분석

**지속적인 개선**

- 최적화 기회를 파악하기 위해 양식 분석의 정기적 검토
- 다양한 양식 디자인과 사용자 흐름의 A/B 테스트
- 반복적인 개선을 위한 사용자 피드백 수집 및 분석
- 업계 표준에 대한 성능 벤치마킹

### 데이터 품질 및 유효성 검사

**스마트한 유효성 검사 전략**

- 즉각적인 사용자 피드백을 위한 실시간 유효성 검사 구현
- 점진적 유효성 검사를 사용하여 복잡한 요구 사항 안내
- 오류 수정 방법을 설명하는 명확한 유효성 검사 메시지 제공
- 사용자 경험 고려 사항과 유효성 검사의 엄격성의 균형

**데이터 무결성**

- 데이터 일관성을 위해 교차 필드 유효성 검사 구현
- 잘못된 데이터를 방지하기 위해 적절한 입력 유형과 제약 조건 사용
- 복잡한 필드에 대한 데이터 포맷의 예와 지침 제공
- 제출된 데이터 품질 및 유효성 검사 효과에 대한 정기 감사
