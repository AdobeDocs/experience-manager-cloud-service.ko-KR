* 이미지 및 비디오 형식에 대한 기본 이해

## Dynamic Media이란 무엇이며, 이를 통해 어떤 도움을 얻을 수 있습니까? {#dm-journey-a}

Dynamic Media은 다양한 시각적 머천다이징 및 마케팅 자산을 On Demand로 제공하는 데 도움이 됩니다. 또한 확대/축소, 360도 회전, 비디오를 포함한 대화형 보기 환경을 만들고 제공하는 데 도움이 됩니다. 자산은 웹, 모바일 및 소셜 사이트에서 소비되도록 동적으로 조정됩니다. 이미지, 비디오 및 3D와 같은 기본 소스 자산 세트를 사용하면 Dynamic Media은 글로벌, 확장 가능 및 성능 최적화 CDN(Content Delivery Network)을 통해 실시간으로 이 풍부한 컨텐츠의 여러 변형을 생성하고 전달합니다.

Dynamic Media은 Adobe Experience Manager Assets 디지털 자산 관리 솔루션의 워크플로우를 통합하여 디지털 캠페인 관리 프로세스를 간소화하고 간소화합니다.

### 무한한 가능성이 있는 하나의 파일

Dynamic Media에 대해 이해할 수 있는 주요 사항 중 하나는 _무한한 가능성이 있는 하나의 기본 자산 파일_.

이 개념을 더 잘 이해하려면 이미지나 비디오와 같은 단일 자산으로 작업하는 방법을 생각해 보십시오. 일반적으로 하나의 기본 자산을 만듭니다. 그런 다음 모든 경험, 필요한 모든 장치, 모든 웹 페이지 및 자산을 사용하는 모든 속성에 대해 동일한 자산의 버전을 수동으로 만듭니다. 시간이 지나면 해당 단일 자산이 버전 기록이 첨부되지 않고 20, 30 이상 버전으로 늘어날 수 있습니다. 여러분이 가지고 있는 모든 이미지나 비디오에 대해 그렇게 한다고 상상해보세요. 자산 버전 수가 스토리지 비용 증가 뿐만 아니라 유지 관리 및 업데이트도 매우 까다로워집니다.

그러나 Dynamic Media은 미디어를 전달하는 데 사용하기 때문에 다른 시스템과 근본적으로 다릅니다 _동적으로_ 단일, 기본 자산 및 URL 호출에서 요청하는 Dynamic Media URL 경로에는 자산이 고객의 화면에 전달될 때 자산을 표시하는 방법을 Adobe 게시 서버에 알려주는 지침이 포함되어 있습니다. 예를 들어 동일한 단일 기본 자산을 사용하면 크기, 형식, 해상도, 무게, 색상, 자르기 및 확대/축소 보기와 같은 효과를 제한 없이 즉시 전달할 수 있습니다.

이 고유한 게재 방법을 사용하면 크기나 대역폭에 관계없이 모든 화면으로 일관된 품질 경험을 보낼 수 있습니다. 또한 전체 크기 비디오는 모든 화면 유형에 맞게 최적화되며 적응적으로 스트리밍되므로 일관된 품질의 사용자 환경을 보장합니다.

_Adobe Dynamic Media을 사용하면 크기나 대역폭에 관계없이 모든 화면에 일관된 품질 경험을 제공할 수 있습니다._

읽으면서 &quot;하나의 기본 자산 파일, 끝없는 가능성&quot;의 개념이 중요한 이유에 대해 더 자세히 알게 될 것입니다.

### 컨텐츠 전달 네트워크

이미지 자산 또는 비디오 자산으로 라이브로 전환할 준비가 되면 강력한 최상위 계층 게재 네트워크로 구성된 Dynamic Media의 백본을 통해 지원됩니다. 그 네트워크는 매일 전 세계 수백 명의 고객을 제공한다. 자산은 Akamai에 의해 호스팅되는 CDN 또는 Content Delivery Network에 배포됩니다. CDN은 최종 사용자에게 컨텐츠, 특히 대형 리치 미디어 컨텐츠를 제공하기 위해 투명하게 협력하는 컴퓨터 서비스 시스템입니다.

CDN 시스템에서 웹 컨텐츠는 인터넷의 웹 캐시에 저장됩니다. 그런 다음 웹 캐시에서 최종 사용자에게 전달하여 빠른 배달을 만듭니다. 따라서 처음 웹 페이지를 다운로드할 때 표시되는 자산은 CDN 캐시에 전달됩니다. 이러한 파일은 서버에 저장되므로 동일한 영역에 있는 사람이 다음 번에 웹 페이지에 액세스할 때 동일한 캐시 콘텐츠가 더 빨리 전달됩니다. 컨텐츠는 최종 사용자에게 가까운 위치에 있으므로 보다 빠르게 전달됩니다. CDN은 웹 페이지 표시를 보다 빠르게 할 수 있도록 하지만, 모든 인스턴스의 중앙 서버가 아닌 캐시 네트워크에서 컨텐츠를 전달하므로 중앙 서버의 대역폭 요구가 줄어듭니다. 최적화된 흐름은 사용자 경험을 향상시켜 매출을 증대시킵니다.

<!-- USE AN IMAGE HERE? ![Content delivery network](/help/assets/assets-dm/cdn.png) -->

지금까지 CDN은 매달 3.5페타바이트의 트래픽을 고객에게 제공합니다. 하루 만에 520억 개의 자산을 전달할 수 있다. 이 숫자는 고객에게 성공적으로 전달된 864,000개의 이미지와 비디오와 같습니다. _매 초_.

### 스마트 이미징

Dynamic Media은 이미 자산을 최적화하고 CDN을 통해 각 자산이 모바일 및 데스크탑 시스템에 빠르게 로드되도록 하는 뛰어난 작업을 수행합니다. 이를 위해 Dynamic Media에서 이미지 사전 설정을 사용하여 이미지 품질을 정의합니다. 또한 전송할 이미지 유형, 선명도 및 기타 여러 경험 또는 페이지의 조각도 정의합니다.

그러나 이미지 사전 설정 외에 Dynamic Media에 값을 더 추가하려면 다음 작업을 수행합니다 _스마트 이미징_.

Smart Imaging은 고객의 브라우저 기능을 기반으로 이미지의 형식과 파일 크기를 자동으로 최적화하여 이미지 자산 전달 성능을 향상시킵니다. 기존 이미지 사전 설정(이미지 사전 설정은 이 여정의 2부에서 논의됨)과 함께 작동하며 전달 시 지능을 사용합니다.

이 인텔리전스는 브라우저 및 네트워크 연결 속도에 따라 이미지 파일 크기를 더 줄입니다. 이미지 자산은 페이지 로드 시간을 대부분을 차지하므로 성능 향상은 다음과 같은 주요 비즈니스 지표에 큰 영향을 줄 수 있습니다.

* 높은 전환
* 사이트에서 보낸 시간
* 낮은 사이트 바운스 비율

전반적으로 스마트 이미징을 사용하면 기존 이미지 사전 설정 설정과 특정 최종 사용자 특성에 따라 22%~47% 성능 향상을 기대할 수 있습니다. 이미지 품질은 마치 전혀 만져지지 않은 것처럼 유지하면서도.

![스마트 이미징](/help/assets/dynamic-media/assets/dm-smart-imaging.png)
_스마트 이미징은 고객의 브라우저 기능과 네트워크 속도에 따라 이미지의 형식과 파일 크기를 자동으로 최적화합니다._

스마트 이미징은 사용자와 Dynamic Media 기술 지원 Adobe 간에 협력하여 작업해야 하므로 기본적으로 켜져 있지 않습니다. 또한, Smart Imaging을 활성화하려면 CDN 캐시를 완전히 지운 다음, 시간에 따라 다시 채워야 합니다. 스마트 이미징을 사용하는 데 관심이 있는 경우 기술 지원 티켓을 제출하여 Adobe을 사용하여 켜지도록 할 수 있습니다. 기술 지원을 통해 미리 스마트 이미징을 시도할 수 있는 URL 매개 변수를 제공합니다. 어떤 웹 페이지나 이미지에서도 사용해 볼 수 있으므로 원하는 성능과 절감율을 얻을 수 있습니다. 그런 다음 전체 사이트에 대해 스마트 이미징을 설정할 수 있습니다.

### 응용 비디오 세트

페이지 또는 기본 페이지에 비디오가 있는 경우 고객은 해당 컨텐츠를 더 오래 사용하여 페이지에 머무르는 경향이 있습니다. 일반적으로 좋은 일입니다. 이 동작은 Adobe이 수행한 분석을 통해 표시되었습니다. 그러나 비디오는 복잡할 수 있습니다. 한 가지 예를 들어, 종종 큰 기본 파일이 있습니다. 비디오 크기를 조정하고 전달하는 방법을 결정하는 것은 복잡하며, 이 모두가 비디오가 보고 있는 장치 및 대역폭에 관계없이 경험이 원활하게 실행되도록 합니다.

이 문제를 해결하기 위해 Dynamic Media은 다음을 생성할 수 있는 기능을 제공합니다 _응용 비디오 세트_.

![응용 비디오 세트](/help/assets/dynamic-media/assets/dm-smart-imaging.png)
_응용 비디오 세트는 다른 비트율 및 형식으로 인코딩된 동일한 비디오의 버전을 그룹화합니다._

먼저 시스템에 업로드하는 원본 기본 비디오로 시작합니다. Dynamic Media 자동 크기 또는 _코드 변환_: 해당 비디오를 여러 비디오로 분할합니다. 그런 다음 제공 시 지능적으로 어떤 비디오 화면, 어떤 품질 및 어떤 포맷을 사용할 것인지 결정하고 휴대폰, 태블릿 또는 데스크탑 컴퓨터로 전달합니다.

예를 들어 iOS 모바일 장치에서 4G, 5G 또는 Wi-Fi와 같은 대역폭을 감지합니다. 그런 다음 응용 비디오 세트 내의 다양한 비디오 비트 전송률 중에서 올바른 인코딩된 비디오를 자동으로 선택합니다. 비디오는 모바일 장치, 태블릿 또는 데스크탑 컴퓨터로 스트리밍됩니다.

또한 네트워크 조건이 변경되면 자동으로 비디오 품질이 전환됩니다. 고객이 데스크탑에서 전체 화면 모드로 전환하면 더 나은 해상도를 사용하여 응답하므로 고객의 보기 경험을 향상시킬 수 있습니다.

응용 비디오 세트를 사용하면 고객이 여러 화면 및 장치에서 Dynamic Media 비디오를 재생할 수 있도록 매끄럽고 고품질 재생이 제공됩니다. 비디오의 복잡성에 대해 이해하면 됩니다.

## Dynamic Media 사용 사례 {#dm-journey-b}

다음은 Dynamic Media에서 긍정적인 고객 참여, 충성도, 전환 및 ROI를 높이는 데 도움이 되는 일반적인 사용 사례 문제 및 솔루션입니다.

### 사용 사례: 기본 파일 접근 방식

Dynamic Media에 가장 중요한 사용 사례 중 하나에도 가장 명백한 것이 있습니다. 즉, 전달되는 페이지 및 경험의 가중치와 컨텐츠 크기(이미지든 비디오이든)가 줄어듭니다.

다음은 일반적인 경험 또는 웹 페이지를 보여줍니다. 페이지의 약 90%는 이미지 및 비디오와 같은 리치 미디어로 구성되며 일반적으로 더 많은 파일이 있습니다.

![컨텐츠 페이지 용량](/help/assets/dynamic-media/assets/dm-content-page-weight.png)
_일반적인 웹 페이지의 컨텐츠 페이지 가중치._

나머지 10%는 HTML, CSS 코드 및 특정 태그입니다. 해당 페이지의 90% 가중치를 최적화하려고 하며 Dynamic Media은 이러한 노력을 지원합니다. 이전에는, _무한한 가능성이 있는 하나의 기본 자산 파일_. 이 방법은 전체 페이지 가중치를 줄이는 데 유용합니다. 기본 자산 하나를 제품 세부 사항 페이지, 축소판 페이지, 장바구니 및 검색 그리드에서 사용하여 사용할 수 있는 기능은 큰 시간이 소요됩니다. 또한 경험 간에 일관성을 보장합니다.

![기본 파일 접근 방식](/help/assets/dynamic-media/assets/dm-onefile.png)
_시계는 하나의 기본 자산 파일이지만 복사본과 같은 다양한 표현물이 즉시 생성됩니다._

Dynamic Media이 하나의 파일과 해당 접근 방식에 대한 일부 솔루션을 사용하여 문제를 해결하는 방법을 자세히 살펴보겠습니다.

| **문제** | **Dynamic Media 솔루션** |
|---|---|
| 모든 자산을 만들고 저장합니다. | 단일 이미지 파일을 사용하여 필요한 렌디션을 제공할 때만 자동으로 만듭니다. |
| 높은 스토리지 비용 | 자산의 여러 사본을 만들고 저장할 필요가 없습니다. |
| 양육권을 유지하는데 어려움이 있다. | 장치에 최적화된 일관된 환경을 제공합니다. |
| 버전 기록이 없습니다. |  |
| 여러 장치에서 브랜드 경험이 일관되지 않습니다. |  |
| 중복된 자산을 생성하는 데 필요한 비용 |  |

한 파일에 대해 생각해 보면 모든 종류의 경험을 위한 자산을 만들 수 있습니다. 하나의 시작 이미지가 있는 경우 해당 이미지의 20, 30 또는 40가지 변형을 만들어야 할 수 있으며, 이 경우 결국 해당 스토리지에 저장하고 비용을 지불해야 합니다.

그런 다음 올바른 이미지를 사용하고 브랜드 간 일관성을 유지하는 기능에 영향을 줄 수 있는지 확인해야 합니다. 그리고 이미지를 찾을 수 없다면 다시 들어가서 해당 자산을 복제해야 합니다.

Dynamic Media을 사용하면 해당 시작 이미지에서 즉시 다양한 이미지를 만들 수 있습니다. 기본 자산으로 제작할 수 있으며 추가 컨텐츠를 만들 때 그래픽 디자인 아티스트나 사진 스튜디오에서 왔다갔다하지 않아도 됩니다. 그리고 그것은 돈과 시간 절약입니다.

하나의 파일 방식을 사용하면 하나의 기본 파일을 사용할 수 있습니다. 그런 다음 고객이 제공하거나 볼 때만 사이트, 속성 및 경험에서 필요한 버전 또는 변환을 만듭니다. 이러한 효율성을 통해 자산에 필요한 스토리지 양이 크게 줄어들고 전체 워크플로우의 복잡성이 줄어듭니다. 또한 Dynamic Media의 전달 시스템을 사용하면 모든 이미지와 비디오가 최적화되고, 빠르게 로드되며, 모든 화면 및 장치에서 잘 보이도록 보장합니다.

### 사용 사례: 비디오

Dynamic Media이 해결하는 다른 사용 사례는 비디오입니다. 비디오가 복잡합니다. 관리가 어렵습니다. 비디오 파일은 고유한 파일 크기 때문에 저장 및 이동하기 어렵습니다.

| **문제** | **Dynamic Media 솔루션** |
|---|---|
| 다양한 장치에 최적화된 비디오를 관리하고 전달하기가 어렵습니다. | 모든 장치의 크기를 자동으로 지정하는 단일 비디오를 사용합니다. |
| 최종 사용자의 사용 가능한 대역폭으로 인해 비디오 지연 또는 재생 품질이 낮습니다. | 사용 가능한 대역폭을 자동으로 감지하고 품질을 조정하여 고품질과 원활한 재생을 보장하는 HTML 플레이어를 통해 비디오를 제공합니다. |
| 모든 버전의 비디오를 수동으로 만드는 것은 불가능하고 시간이 많이 걸립니다. 이렇게 하면 장치 간에 원활하게 표시되고 재생될 수 있습니다. | 간소화된 워크플로우를 통해 번거로운 코드 변환 작업을 수행할 수 없습니다. |
|  | 더 높은 가치 작업을 위해 여유 시간 |

고객은 다음과 같은 문제를 해결하려고 Dynamic Media에 옵니다.

&quot;_우리는 비디오를 가지고 있고, 그것을 만드는 데 많은 돈을 썼습니다. 그러나 우리는 그것을 페이지에 놓거나 전달하는 것을 피했습니다. 왜냐하면 우리의 시험으로 비디오의 품질을 보장할 수 없거나 실제로 재생될 수 있을지 확신할 수 없기 때문입니다. 그리고 궁극적으로, 이것은 브랜드와 잠재적으로 전환까지 우리의 역할에 영향을 줍니다._&quot;

Dynamic Media의 솔루션은 기본 비디오 파일 하나를 가져와서 Dynamic Media이 코드 변환 프로세스를 통해 모든 크기를 만들도록 하는 것입니다. 그런 다음 Dynamic Media의 지능형 비디오 플레이어와 결합합니다. 이 워크플로우는 기본 랜딩 페이지나 카테고리 또는 제품 세부 사항 페이지에서 해당 비디오를 사용하는지 여부에 관계없이 전체 단계에서 일관되도록 하고 높은 품질과 함께 배달되도록 보장합니다.

고려해야 할 몇 가지 추가 사용 사례가 있습니다.

### 사용 사례: 단일 진리

| **문제** | **Dynamic Media 솔루션** |
|---|---|
| 조직 전체에 분산되어 있는 디지털 자산은 서로 다른 팀 또는 비즈니스 단위로 분산됩니다. | 모든 디지털 자산을 중앙 위치에 저장하고 관리합니다. |
| 팀 구성원이 로컬 버전을 다운로드하여 만듭니다. | 팀 구성원은 단일 기본 파일을 사용하여 _및_ 다양한 화면 크기 및 장치에 필요한 모든 버전을 제공합니다. |
| 모든 경험 및 장치에 대해 만들어진 단일 사용 자산 | 자산을 한 번 사용하지 않고 시간과 비용을 절약합니다. |

### 사용 사례: 리치 미디어를 위한 AI 기반의 스마트 자르기

| **문제** | **Dynamic Media 솔루션** |
|---|---|
| 이미지 또는 비디오를 수동으로 그리기, 측정 및 잘라내어 모든 화면 크기 및 장치에서 초점을 맞추고 적절하게 표시하는 데 많은 시간과 노력이 소요됩니다. | Adobe Sensei AI 기능인 Dynamic Media의 스마트 자르기를 사용하여 모든 이미지나 비디오에서 초점을 자동으로 감지하고 자르면 유지 관리할 수 있습니다. |
| 영향을 많이 받는 경험을 만드는 데 더 잘 사용될 수 있는 손실된 시간입니다. | 화면 크기에 상관없이 원하는 관심 영역을 캡처합니다. |
| 모든 경험 및 장치에 대해 만들어진 단일 사용 자산 | 번거로운 수동 작업을 제거하고 모든 장치나 화면에서 잘 보이는 고품질의 빠른 로드 이미지와 비디오를 제공합니다. |

### 사용 사례: 대화형 미디어 작성

| **문제** | **Dynamic Media 솔루션** |
|---|---|
| 참여하지 않고, 충성도를 렌더링하거나, 전환을 유도하지 않는 플랫 및 정적 고객 경험. | 기술 전문가가 아닌 사용자가 보다 동적이고 매력적인 경험을 위해 핫 스팟, 회전 메뉴 및 스핀 세트와 같은 대화형 요소를 쉽고 원활하게 추가할 수 있습니다. |

## Dynamic Media 시스템을 통해 자산이 이동하는 방법 {#dm-journey-c}

다음은 Dynamic Media에 대한 일반적인 워크플로우를 보여줍니다.

![Dynamic Media 워크플로우](/help/assets/dynamic-media/assets/dm-workflow.png)
_자산이 Dynamic Media 시스템을 통해 이동하는 방법입니다._

기본 자산을 끝에 배치하기 위한 주요 목표로 만들기 단계부터 시작합니다. 이러한 기본 자산은 사진 촬영, 비디오 공급업체 또는 사용자가 만든 오디오 파일일 수 있습니다. Adobe InDesign, Adobe Photoshop, Adobe Illustrator과 같은 Adobe의 Creative Suite 애플리케이션을 사용하여 컨텐츠 작업을 지원할 수 있습니다.

만들기 부분이 완료되면 자산을 Dynamic Media에 업로드하여 자산을 작성 솔루션에 넣습니다. Dynamic Media 내에서 사이트의 다양한 웹 페이지에 대해 올바른 이미지 사전 설정 및 뷰어가 연결되어 있는지 확인합니다.

궁극적으로, 모든 콘텐츠를 최적화하고 Dynamic Media 서버에 게시하여 웹, 인쇄, 이메일, 데스크탑 및 모바일 장치에서 사용할 수 있도록 합니다.

### Dynamic Media에 자산 업로드

기본 자산 만들기를 마치면 Dynamic Media에 업로드합니다. 업로드하는 파일의 유형, 파일의 형식 및 크기는 Dynamic Media의 중요한 속성입니다. 한 파일 지원에서 최대 값을 가져올 수 있도록 업로드할 때가 있습니다.

예를 들어, 아래 감시 이미지는 4560 x 3020픽셀입니다. 또한 이 크기의 이미지를 사용할 수는 없지만 계속 업로드할 수 있습니다. 이미지가 클수록 Dynamic Media이 제공할 수 있는 품질이 향상되며 축소판 표현까지 제공됩니다. 기억: 당신은 쉽게 _감소_ 기존 이미지의 해상도. 하지만 만약 _증가_ 영상의 결제는 만족스럽지 않을 것 같다.

![Dynamic Media에 업로드할 권장 형식](/help/assets/dynamic-media/assets/dm-upload-formats.png)
_자산 업로드에 대한 고려 사항입니다._

Adobe은 자산을 무손실 형식으로 업로드하는 것을 권장합니다. 일반적으로 JPEG을 제공할 때 또는 JPEG을 계속 저장할 때 시간이 지남에 따라 이미지 품질이 떨어지기 때문에 JPEG을 피하는 것이 좋습니다. 사용 가능한 손실 없는 포맷의 최고 해상도 이미지로 시작하려고 합니다. 이 형식은 일반적으로 TIFF 또는 PNG 파일입니다.

색상 공간의 경우 디지털 채널이나 웹 보기를 생각할 때 일반적으로 RGB(빨강, 녹색, 파랑)를 생각할 수 있습니다.

대부분의 경우 CMYK로 무언가를 전달하는 것 또는 CMYK로 전달하는 것에 대해 생각하지 않습니다. 그 이유는 컬러 공간이 인쇄된 물건들을 배달하는데 가장 많이 사용되기 때문입니다. 하지만 Dynamic Media은 두 색상 공간에서 모두 게재할 수 있습니다.

창고 도매 클럽과 같이 여전히 인쇄를 하는 손님들이 많이 있습니다. 그리고 매주 전단지를 인쇄하는 식료품점도 있습니다. 이러한 고객은 이미지가 두 색상 공간에 있어야 합니다. 일반적으로 두 개의 서로 다른 이미지가 필요합니다. CMYK의 RGB 하나와 CMYK의 하나 그러나 CMYK 자산을 Dynamic Media에 직접 업로드하고, Dynamic Media에서 이미지 사전 설정을 통해 또는 색상 프로파일을 통해 RGB 자산을 자동으로 전달하도록 할 수 있습니다. 여러 버전의 파일을 만들 필요가 없으므로 _무한한 가능성이 있는 하나의 기본 자산 파일_.

### 자산 게시 및 미리 보기

자산을 Dynamic Media에 업로드한 후에는 다음 작업을 수행하는 것이 좋습니다 _게시_ 자산을 선택한 다음 **[!UICONTROL 게시]** 또는 **[!UICONTROL 빠른 게시]** Dynamic Media. 경험에서 자산을 사용하려는 경우 자산 게시가 필요합니다. 자산이 게시되면 복사한 Dynamic Media 생성 URL을 사용하여 웹 페이지에 포함하거나, 페이지에 코드를 포함함으로써 사용할 수 있습니다.

자산을 수동으로 게시하는 것 외에 업로드 시 사용자 개입 없이 자산을 즉시 게시하도록 Dynamic Media을 구성할 수 있습니다.
