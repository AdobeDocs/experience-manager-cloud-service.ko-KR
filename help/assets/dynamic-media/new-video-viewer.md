# Dynamic Media의 새 비디오 뷰어 {#new-video-viewer-dynamic-media}

Dynamic Media용 새 비디오 뷰어는 Adobe Experience Manager(AEM)에서 현대화된 비디오 재생 환경을 제공합니다. 기존 Dynamic Media 워크플로우로 계속 작업하면서 작성, 미리보기 및 Sites 환경 전반에서 일관되고 확장 가능한 보기 환경을 제공합니다.

Dynamic Media의 기존 비디오 뷰어는 코어 재생 요구 사항을 지원하지만 최신 분석 및 통합 시나리오에 대해 제한된 확장성 및 이벤트 수준 통합을 제공합니다

새 비디오 뷰어는 다음과 같은 방법으로 이러한 제한 사항을 해결합니다.

* 보다 일관된 재생 경험 제공
* 명시적 뷰어 선택 허용
* 프로그래밍 방식 소비를 위해 구조화된 재생 이벤트 내보내기
* 외부 분석 및 외부 시스템과의 통합 지원

뷰어는 추가 옵션으로 사용할 수 있으며 지원되는 경우 명시적으로 선택해야 합니다. 기존 비디오 뷰어를 자동으로 대체하지 않습니다.

새로운 비디오 뷰어는 기존 구현을 중단하지 않고 확장 가능한 향상된 비디오 경험을 필요로 하는 조직을 위한 것입니다.

> **참고**
>
> 새 비디오 뷰어는 제한된 가용성 기능입니다. [지원 티켓](https://helpx.adobe.com/kr/enterprise/using/support-for-experience-cloud.html)을 만들어 활성화할 수 있습니다.


## 새 비디오 뷰어 작동 방식 {#how-it-works}

새 비디오 뷰어는 다음과 같이 작동합니다.

1. 비디오 자산은 Dynamic Media와 동기화된 폴더에 수집됩니다.
2. 자산 세부 정보 페이지에서 **비디오(신규)**&#x200B;를 사용하여 비디오를 미리 볼 수 있습니다.
3. Sites 페이지를 작성할 때 **Dynamic Media** 구성 요소에서 새 비디오 뷰어를 선택할 수 있습니다.
4. 재생 중에 뷰어가 구조화된 이벤트를 상위 창으로 내보냅니다.
5. 선택적 뷰어 수정자를 사용하여 재생 동작을 제어할 수 있습니다.

## 기존 비디오 뷰어와의 주요 차이점 {#key-differences}

| 영역 | 설명 |
|------|-------------|
| 뷰어 가용성 | 이름이 **비디오(새 비디오)**&#x200B;인 새 옵션으로 나타납니다. |
| 뷰어 선택 | 명시적으로 선택해야 함 |
| 확장성 | 구조화된 재생 이벤트 내보내기 |
| 통합 | 기존 Dynamic Media 워크플로우에서 계속 작동합니다 |

## 사전 요구 사항 {#prerequisites}

새 비디오 뷰어를 사용하기 전에 다음 전제 조건이 충족되는지 확인하십시오.

| 요구 사항 | 설명 |
|------------|-------------|
| Dynamic Media 동기화 | 자산 폴더를 Dynamic Media와 동기화해야 합니다. |
| 비디오 프로필 | 비디오 프로필을 폴더에 적용해야 합니다. |
| 비디오 자산 | 비디오를 폴더로 수집해야 합니다. |

새 비디오 뷰어는 **AEM as a Cloud Service 버전 2025.7.0**&#x200B;부터 사용할 수 있습니다.

새 비디오 뷰어를 활성화하거나 비활성화하려면 Adobe 고객 지원 센터에 문의하십시오.

## 새 비디오 뷰어 미리 보기 {#preview}

다음 단계를 실행하여 자산 세부 사항 페이지에서 새 비디오 뷰어를 미리 봅니다.

1. **Assets** > **파일**(으)로 이동하여 비디오 자산이 포함된 폴더를 엽니다.
2. 비디오 자산을 클릭하여 자산 세부 사항 페이지를 엽니다.
3. 왼쪽 패널에서 **뷰어**&#x200B;를 클릭합니다.
4. **뷰어** 패널에서 **비디오(새 비디오)**&#x200B;을(를) 선택합니다.
5. 미리 보기 링크를 복사하려면 **URL**&#x200B;을(를) 클릭하십시오.
   ![URL 복사](assets/Copy-url1.jpg)

## Sites에서 새 비디오 뷰어 사용 {#use-in-sites}

새 비디오 뷰어는 AEM Sites의 기존 **Dynamic Media** 구성 요소를 통해 사용할 수 있습니다.

### Dynamic Media 구성 요소 추가

Dynamic Media 구성 요소를 사용하여 비디오를 추가하려면 다음 단계를 수행하십시오.

1. **사이트 편집기**&#x200B;에서 페이지를 엽니다.
2. **Dynamic Media** 구성 요소를 페이지의 필요한 위치로 드래그합니다.
3. 페이지에서 **Dynamic Media** 구성 요소를 선택하십시오.
4. 구성 요소를 클릭하여 자산 선택기를 엽니다.
5. 비디오 자산을 선택합니다.

![Dynamic Media 구성 요소 드래그](assets/drag-component.jpeg)

### 뷰어 구성

다음 단계를 실행하여 뷰어 사전 설정을 구성합니다.

1. 페이지에서 **Dynamic Media** 구성 요소를 선택하십시오.
2. 구성 요소 도구 모음에서 **구성**&#x200B;을 클릭합니다.
   ![Dynamic Media 설정 열기](assets/configure-asset.png)

3. **Dynamic Media 설정 대화 상자**&#x200B;의 **뷰어 사전 설정** 드롭다운 목록에서 **비디오(새 비디오)**&#x200B;을(를) 선택하십시오.
   ![비디오(새) 뷰어 사전 설정 선택](assets/viewer-preset.jpeg)

4. **뷰어 수정자** 필드에 필요한 수정자를 입력합니다(예: `autoplay=true&muted=true`).
   ![뷰어 수정자](assets/additional-modifiers.jpeg)

5. 변경 사항을 저장합니다.

새 비디오 뷰어를 사용하여 비디오가 페이지에 로드됩니다.

> **참고:** 새 비디오 뷰어가 기존 비디오를 자동으로 대체하지 않습니다. 사용자는 Dynamic Media 구성 요소를 사용할 때 **뷰어 사전 설정**&#x200B;에서 수동으로 **비디오(새 비디오)**&#x200B;을(를) 선택하거나 필요한 경우 새 비디오 뷰어를 가리키도록 직접 URL을 업데이트해야 합니다.

### 직접 URL을 사용하여 비디오 마이그레이션

Dynamic Media 구성 요소 대신 직접 URL을 통해 비디오에 액세스하는 경우 URL을 업데이트하여 새 비디오 뷰어로 전환할 수 있습니다. 예를 들어`https://s7d1.scene7.com/dmviewers/html5/VideoViewer.html?asset=<video-asset>`

## 뷰어 수정자 {#viewer-modifiers}

뷰어 수정자를 사용하여 에셋 로드, 재생 동작, 스트리밍 형식 선택 및 뷰어 프레젠테이션을 제어할 수 있습니다.

| 수정자 | 설명 |
|--------|-------------|
| `asset` | 비디오 또는 응용 비디오 세트의 자산 ID를 지정합니다. |
| `posterimage` | 재생이 시작되기 전에 표시되는 이미지를 지정합니다. |
| `serverurl` | 이미지 제공 루트 경로를 지정합니다. |
| `contenturl` | 콘텐츠 루트 경로를 지정합니다. |
| `videoserverurl` | 비디오 서버 루트 경로를 지정합니다. |
| `sources.dash` | 재생할 DASH 매니페스트 URL을 지정합니다. |
| `sources.hls` | 재생할 HLS 매니페스트 URL을 지정합니다. |
| `autoplay=true` | 비디오가 로드되면 자동으로 재생을 시작합니다. |
| `controls=true/false` | 비디오 재생 컨트롤을 표시하거나 숨깁니다. |
| `loop=true` | 비디오가 종료되면 자동으로 재생을 다시 시작합니다. |
| `muted=true` | 음소거 상태에서 재생을 시작합니다. |
| `playbackrates` | 사용 가능한 재생 속도 옵션을 지정합니다. |
| `playback` | 스트리밍 형식(자동, hls, 대시 또는 점진적)을 지정합니다. |
| `progressivebitrate` | 점진적 재생을 위한 비트율을 지정합니다. |
| `initialbitrate` | 적응형 스트리밍에 대한 초기 비트율을 지정합니다. |
| `isletterboxed=true/false` | 비디오가 레터박스 상태인지 또는 스트레치 상태인지 여부를 제어합니다. |
| `customcss` | 뷰어 스타일을 지정할 사용자 지정 CSS 파일을 지정합니다. |
| `transition` | 뷰어 컨트롤의 전환 동작을 표시하거나 숨깁니다. |

**뷰어 한정자** 필드에 쿼리 매개 변수로 한정자가 지정되었습니다.

## 지원되는 이벤트 {#supported-events}

재생 중에 새 비디오 뷰어에서 다음 이벤트가 발생합니다.

| 이벤트 유형 | 설명 |
|-----------|-------------|
| play | 비디오 재생 시작 |
| pause | 비디오가 일시 중지되었습니다. |
| 찾기 | 사용자가 비디오 내에서 찾기 |
| 로드 | 비디오가 로드됨 |
| 닫기 | 플레이어가 닫힘 |
| 메타데이터 | 지속 시간과 같은 메타데이터 |
| 이정표 | 재생 마일스톤 도달 |
| current_time | 주기적 재생 위치 |
| 전체 화면 | 전체 화면으로 전환 |
| 전체 화면 해제(_F) | 전체 화면 종료 |

## 상위 창에서 이벤트 처리 {#handling-events}

새 비디오 뷰어는 비디오 상호 작용 중에 재생 관련 메시지를 상위 페이지에 보냅니다.

이러한 이벤트를 처리하려면 상위 애플리케이션이 브라우저 메시지 이벤트를 수신하고 데이터를 처리하기 전에 메시지 출처를 확인해야 합니다.

이벤트 페이로드는 이벤트 타입, 재생 상태, 현재 재생 시간, 추가 메타데이터 등의 정보를 포함한다. 이러한 이벤트는 분석 추적, 사용자 지정 상호 작용 또는 외부 시스템과의 통합을 지원하는 데 사용할 수 있습니다.

Adobe은 메시지 원본을 확인하여 신뢰할 수 있는 Dynamic Media 도메인에서만 이벤트가 처리되도록 할 것을 권장합니다.

## 새 비디오 뷰어에 대한 비디오 참여 보고서 {#video-engagement-report}

비디오 참여 보고서는 Dynamic Media의 새 비디오 뷰어를 사용하여 재생되는 비디오에 대한 분석 지표를 제공합니다. 이 보고서는 지정된 달에 대한 집계된 성능 데이터를 제공하며 월별 보고를 지원합니다.

보고서는 요청 시 생성됩니다. 보고서를 요청하려면 [지원 티켓](https://helpx.adobe.com/kr/enterprise/using/support-for-experience-cloud.html)을 만들고 다음 세부 정보를 제공하세요.

* 보고서 월 - 보고서가 필요한 월을 지정합니다(예: 2026년 1월).
* 게재 이메일 주소 - 보고서를 게재할 그룹(권장) 또는 개인의 이메일 주소

이 보고서는 보기 수, 노출 횟수, 시청 시간, 완료율 및 참여 점수를 포함한 비디오당 참여 지표를 제공합니다.

### 보고서 형식

* 보고서는 CSV 형식으로 전달됩니다.
* 각 행은 단일 비디오를 나타냅니다.
* 선택한 보고 기간 동안 지표가 집계됩니다.
* 삭제된 자산은 보고서에서 제외됩니다.
* `tenant_name`별 필터링을 지원합니다.

### 보고서 필드

비디오 참여 보고서에는 다음 필드가 포함됩니다.

| 필드 | 설명 | 계산 |
|-------|------------|-------------|
| `video_id` | 고유 비디오 식별자. | NA |
| `video_name` | 비디오 에셋의 이름입니다. | NA |
| `video_created_date` | 비디오가 생성된 날짜. | NA |
| `duration_in_seconds` | 비디오 지속 시간(초)입니다. | NA |
| `video_views` | 선택한 보고 기간 동안의 총 비디오 재생 이벤트 수입니다. | NA |
| `video_impressions` | 비디오가 로드된 총 횟수입니다. | NA |
| `video_watched_seconds` | 모든 재생 이벤트에서 시청한 총 시간(초)입니다. | 모든 재생 이벤트에서 시청한 시간(초) 합계 |
| `play_rate` | 비디오 로드와 관련된 비디오 재생의 비율입니다. | (`video_views` ÷ `video_impressions`) × 100 |
| `avg_time_watched_in_seconds` | 보기당 평균 시청 시간(초)입니다. | `video_watched_seconds` ÷ `video_views` |
| `avg_completion_rate` | 전체 비디오 완료에 도달한 보기의 비율입니다. | (완료된 보기 ÷ `video_views`) × 100 |
| `engagement_score` | 모든 재생 이벤트에서 평균 시청 비율입니다. | (`video_views`÷ 모든 세션에서 본 비디오 타임라인의 총 비율) |
| `tenant_name` | 데이터와 연결된 회사 또는 테넌트의 식별자입니다. | NA |

## 자주 묻는 질문 {#faq-video-engagement}

+++비디오가 자동 재생으로 설정된 경우 자동으로 보기로 계산됩니까? 아니면 사용자가 최소 기간 동안 시청한 후에만 보기로 계산됩니까?

자동 재생은 비디오 보기로 계산됩니다. 자동으로 시작된 재생은 보기로 기록됩니다.

+++

+++사용자가 비디오의 일부만 시청하는 경우(예: 10초 분량의 처음 2초 및 마지막 2초) 완료된 보기로 계산됩니까?

비디오 일부를 건너뛴 경우에도 재생이 비디오 타임라인의 끝에 도달하면 보기가 완료된 것으로 카운트됩니다.

+++

+++사용자가 뒤로 이동하여 비디오의 일부를 다시 시청하는 경우 video_views 카운트, engagement_score 또는 둘 다 증가합니까?

비디오의 일부를 다시 시청해도 video_views 카운트가 증가하지 않습니다. 추가 재생은 engagement_score에 기여합니다.

+++

+++동일한 사용자가 페이지를 다시 로드하지 않고 동일한 비디오를 여러 번 시청하는 경우 video_views 및 engagement_score는 어떻게 계산됩니까?

페이지를 다시 로드하지 않고 반복해서 재생해도 video_views 카운트가 증가하지 않습니다. 추가 재생은 engagement_score에 기여합니다.
