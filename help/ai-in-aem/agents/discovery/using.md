# 검색 에이전트 {#discovery-agent}

검색 에이전트는 간소화되고 클릭되지 않는 검색 환경을 위해 자연스럽고 대화식 프롬프트를 통해 AEM 콘텐츠를 온디맨드로 제공합니다. Assets, 컨텐츠 조각 및 적응형 Forms 전반에서 지능적으로 검색하여 이미지, 비디오, PDF 문서, 문서 및 양식 템플릿과 같은 관련 자료를 제공합니다. 자연어를 사용하면 AEM Assets 인터페이스에서 복잡한 쿼리를 작성하거나 필터를 적용하지 않고도 콘텐츠를 검색할 수 있습니다. 에이전트는 프롬프트에 따라 조정된 결과를 에셋 메타데이터 및 게재 URL과 함께 반환하여 다른 애플리케이션에 임베드할 수 있도록 준비됩니다.

검색 에이전트의 몇 가지 주요 이점은 다음과 같습니다.

* **통합 콘텐츠 검색**: 하나의 대화 인터페이스에서 이미지, 비디오, PDF 문서, 문서 및 양식과 같은 모든 유형의 AEM 콘텐츠에 액세스합니다.

* **더 빠른 캠페인 계획**: 이메일, 웹 및 소셜 채널에서 마케팅 캠페인에 대한 시각적 개체 및 양식을 빠르게 수집합니다.

* **생산성 향상**: 자동화된 인텐트 기반 검색을 통해 저장소를 탐색하거나 메타데이터를 필터링하는 데 드는 시간을 줄입니다.

* **일관된 콘텐츠 사용률**: 승인된 에셋과 조각을 재사용하여 채널 전반에서 브랜드 일관성을 유지할 수 있습니다.

## 기술 {#skills-discovery-agent}

검색 에이전트는 다음과 같은 기술을 제공합니다.

* **자연어 콘텐츠 검색**\
  검색 에이전트를 사용하면 사용자가 복잡한 검색 쿼리가 필요하지 않고 간단한 자연어 프롬프트를 사용하여 Adobe Experience Manager(AEM) 내에서 관련 에셋, 콘텐츠 조각 및 적응형 양식을 찾을 수 있습니다.

* **태그 기반 자산 검색**

  검색 에이전트는 자연어 프롬프트를 사용하여 AEM 저장소의 특정 태그와 연결된 에셋을 찾으므로 사용자가 조직의 분류법에 따라 구성되거나 구성되지 않은 콘텐츠에 빠르게 액세스할 수 있습니다.

* **폴더 기반 콘텐츠 검색:**\
  검색 에이전트는 AEM의 폴더 이름을 참조하는 자연어 프롬프트를 해석하여 에셋을 식별할 수 있습니다. 사용자는 저장소를 수동으로 탐색하지 않고 프롬프트에서 폴더를 언급하기만 하면 되므로 적절한 콘텐츠를 찾는 데 필요한 클릭 수를 크게 줄일 수 있습니다.

## 가상 사용자 {#personas-discovery-agent}

### 캠페인 관리자 {#campaign-managers}

Discovery Agent를 사용하면 Campaign 관리자는 신뢰할 수 있고 성능이 뛰어난 컨텐츠를 신속하게 식별하여 재사용할 수 있습니다.

### 채널 마케터 {#channel-marketers}

검색 에이전트를 사용하면 채널 마케터가 관련 에셋을 효율적으로 찾아 응집력 있는 다중 채널 경험을 만들 수 있습니다.

### DAM 라이브러리 관리자 {#dam-librarians}

DAM Librarians는 조직에서 설정한 메타데이터 표준이 누락된 에셋에 플래그를 지정하여 일관된 거버넌스를 지원하고 에셋이 채널 간에 완전하고 사용 가능한 상태를 유지하도록 할 수 있습니다.

### 에이전시 및 파트너 {#agencies-partners}

에이전시와 파트너는 Content Hub 내에서 브랜드가 승인한 자산을 쉽게 찾아 재사용하여 브랜드 표준에 맞게 유지하면서 크리에이티브 작업을 가속화할 수 있습니다.

## 일반적인 사용 사례 및 샘플 프롬프트 {#use-cases-prompts}

### 자산 {#discovery-agent-use-cases-assets}

**태그 기반 자산 검색**

검색 에이전트는 자연어 프롬프트를 사용하여 AEM 저장소의 특정 태그와 연결된 에셋을 찾으므로 사용자가 조직의 분류법에 따라 구성된 콘텐츠에 빠르게 액세스할 수 있습니다.

샘플 프롬프트:

`office` 폴더에 `WKND` 태그가 지정된 이미지를 표시합니다.

**폴더 기반 콘텐츠 검색:**\

샘플 프롬프트:

* `WKND` 폴더에 svgs가 있습니까?
* `Nov 1 2025` 폴더에 `WKND` 이후에 수정된 자산을 표시합니다.
* `lifestyle` 폴더에 `WKND`개의 이미지를 나열합니다.

**해결 방법 및 형식 기반 에셋 검색**

Discovery Agent는 파일 형식이나 최소 해상도와 같은 특정 품질 요구 사항을 충족하는 에셋을 식별할 수 있으므로, 채널 간에 고품질 서비스를 제공하고 재사용할 수 있는 제품 비주얼을 신속하게 찾을 수 있습니다.

샘플 프롬프트:

**방향 기반 콘텐츠 검색**

검색 에이전트는 사람 존재 및 이미지 방향과 같은 시각적 속성을 인식하여 자산을 필터링할 수 있습니다. 이렇게 하면 사용자가 AEM에서 여러 필터를 수동으로 적용하지 않고도 가장 관련성이 높은 시각화로 콘텐츠를 빠르게 좁힐 수 있습니다.

샘플 프롬프트:

가로 방향으로 사용자와 에셋을 표시합니다.

### 콘텐츠 조각 {#discovery-agent-use-cases-content-fragments}

검색 에이전트는 캠페인 이름, 제품 브랜드, 게시 상태 및 최근 만들기 활동에 대한 자연어 참조를 해석하여 사용자가 올바른 콘텐츠 조각을 빠르게 찾을 수 있도록 지원합니다. 이를 통해 팀은 AEM에서 폴더를 수동으로 탐색하거나 여러 필터를 적용하지 않고도 캠페인 준비 조각을 표시하고 브랜드별 콘텐츠를 볼 수 있습니다.

샘플 프롬프트:

* WKND 오퍼 캠페인을 만들기 위한 콘텐츠 조각을 표시합니다.

* 아메리카노 음료에 대한 컨텐츠 조각을 표시합니다.

* WKND 음료에 대해 게시된 모든 콘텐츠 조각을 표시합니다.

* 지난 2주 동안 만든 모든 콘텐츠 조각을 나열합니다.

### 양식 {#discovery-agent-use-cases-forms}

검색 에이전트를 사용하면 자연어 프롬프트를 사용하여 적응형 양식을 빠르게 찾을 수 있습니다. 양식 콘텐츠 및 메타데이터를 검색하여 프롬프트의 키워드를 기반으로 일치하는 항목을 찾습니다. 즉, 검색어가 양식 제목이나 설명에 없는 경우에도 관련 양식을 성공적으로 검색할 수 있습니다.

샘플 프롬프트:

* 대출 신청서 양식 모두 보여 주세요.
* 작업에 적용할 양식을 찾습니다.
* 연락처 양식을 찾습니다.
* 직원 온보딩 양식을 찾고 있습니다.
* 신용카드 신청서 좀 보여주세요

참고: 양식 검색은 현재 Edge Delivery Services 양식만 지원하며 현재 양식에는 태그 기반 검색을 사용할 수 없습니다.

## 검색 결과 {#discovery-agent-search-results}

### 자산 {#discovery-agent-search-results-assets}

검색 에이전트는 정확한 일치가 먼저 나타나도록 각 쿼리에 대해 관련성을 기준으로 정렬된 상위 20개의 결과를 반환합니다. 에이전트는 메타데이터 기반 쿼리를 시맨틱 검색과 결합하여 일치할 가능성이 있는 집중 세트를 조합한 다음 LLM을 사용하여 사용자 의도에 따라 등급을 매깁니다. 이 혼합 접근 방식은 직접 키워드 일치에 전적으로 의존하지 않고 정확하고 컨텍스트 인식 결과를 제공합니다.

각 결과에는 자산 경로, 작성자, 생성 날짜, 제목, 설명, 형식, 마지막 수정자, 마지막 수정 날짜, 파일 크기, 차원, [Dynamic Media URL](/help/assets/dynamic-media/dynamic-media.md) 및 관련 태그와 같은 주요 자산 메타데이터와 함께 자산 이름이 포함됩니다. 에셋이 승인됨 상태인 경우 결과에 OpenAPI URL이 있는 [Dynamic Media](/help/assets/dynamic-media-open-apis-overview.md)도 포함됩니다.

에셋 경로를 클릭하여 AEM 내의 에셋 위치로 원활하게 이동할 수 있습니다.

![검색 에이전트를 사용하여 자산 검색](/help/ai-in-aem/agents/discovery/assets/search-results-discovery-agent.png)

이러한 세부 정보를 보려면 각 에셋으로 이동하지 않고도 이러한 에셋 세부 정보를 사용하여 에셋이 요구 사항을 충족하는지 신속하게 평가할 수 있습니다.

>[!NOTE]
>
>[Dynamic Media URL](/help/assets/dynamic-media/dynamic-media.md) 필드는 자산이 게시되고 유효한 Dynamic Media 라이선스가 있는 경우에만 검색 결과에 표시됩니다. 마찬가지로 [OpenAPI URL이 있는 Dynamic Media](/help/assets/dynamic-media-open-apis-overview.md) 필드는 유효한 Dynamic Media 라이선스가 있고 OpenAPI가 있는 Dynamic Media가 AEM as a Cloud Service 인스턴스에 대해 활성화되어 있는 경우에만 표시됩니다.

### 콘텐츠 조각 {#discovery-agent-search-results-content-fragments}

검색 에이전트는 콘텐츠 조각에 대한 전체 텍스트 검색 기능을 제공하여 지정된 프롬프트와 가장 일치하는 상위 20개 결과를 반환합니다. 각 결과에는 콘텐츠 조각 경로, 작성자, 생성 날짜, 변형, 마지막 수정자 및 마지막 수정 날짜 필드와 같은 주요 메타데이터 필드와 함께 콘텐츠 조각 이름이 포함됩니다.

![검색 에이전트를 사용하여 콘텐츠 조각 검색](/help/ai-in-aem/agents/discovery/assets/search-content-fragments-discovery-agent.png)

콘텐츠 조각 경로를 클릭하여 AEM 내의 콘텐츠 조각 위치로 원활하게 이동할 수 있습니다.

## 프롬프트 우수 사례 {#prompting-best-practices-discovery-agent}

에이전트가 정확하고 적절한 결과를 반환할 수 있도록 자연어 프롬프트에 간결한 세부 정보를 지정합니다. 찾고 있는 내용을 명확하게 설명할수록 에이전트는 출력을 더 잘 다듬고 좁힐 수 있습니다. 예를 들어 다음 작업을 수행할 수 있습니다.

* 프롬프트에서 태그, 폴더 이름, 작성 날짜, 게시 상태, 작성자 이름과 같은 자산 메타데이터를 정의하여 자산을 필터링합니다.

* 카테고리(운동화, 전자 제품), 시즌(가을, 봄), 이벤트(블랙 프라이데이, 제품 출시) 및 채널(웹, 이메일, 인쇄)과 같은 조직별 메타데이터를 사용하여 콘텐츠를 추가로 필터링합니다.
