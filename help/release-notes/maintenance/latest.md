# 유지 관리 릴리스 정보 {#maintenance-release-notes}

다음 섹션에서는 Experience Manager as a Cloud Service의 현재 유지 관리 릴리스에 대한 기술 릴리스 정보를 간략히 소개합니다.

## 릴리스 25194 {#25194}

다음은 2026년 4월 1일에 공개적으로 릴리스된 유지 보수 릴리스 25194에 대한 지속적인 개선 사항을 요약합니다. 이전 유지 관리 릴리스는 릴리스 24678.

2026.4.0 기능 활성화는 이 유지 관리 릴리스에 대한 전체 기능 세트를 제공합니다. 자세한 내용은 [Experience Manager 릴리스 로드맵](https://experienceleague.adobe.com/ko/docs/experience-manager-release-information/aem-release-updates/update-releases-roadmap)을 참조하십시오.

>[!NOTE]
>
>릴리스 24893이 비공개로 설정되었습니다.

### 개선 사항 {#enhancements-25194}

* ASSETS-65127: 이벤트 사용자 지정 메타데이터: 메타데이터 이름 처리가 개선되었습니다.
* ASSETS-63313: C2PA 매니페스트를 기반으로 내보낸 에셋 및 상위 항목에 대한 관련 링크를 자동으로 만듭니다.
* ASSETS-10995: 다운로드 zip의 에셋 수 제한.
* FORMS-24388: 개발자가 공유 서버에 의존하지 않고 구성을 빌드하고 테스트할 수 있는 대화형 통신(IC) 편집기의 로컬 개발 환경을 추가했습니다. 이러한 향상된 기능을 통해 기업 고객은 더 빠르게 반복하고 환경 의존성을 줄이며 전반적인 개발 생산성을 향상시킬 수 있습니다.
* FORMS-24014: &quot;AND&quot; 논리를 사용하여 결합 조건을 지원하도록 파일 첨부 구성 요소에 대한 규칙 편집기가 개선되었습니다. 예를 들어, &quot;첨부 파일이 변경되고 패널이 유효한 경우 이 작업을 수행하십시오.&quot;와 같은 규칙을 허용할 수 있습니다. 이전에는 첨부 파일에 추가 조건을 사용할 수 없었습니다. 이 업데이트를 통해 보다 복잡한 규칙 정의를 사용하여 고급 워크플로를 지원할 수 있었습니다.
* FORMS-23571: 사용자 지정 이벤트 외에 기본 제공(OOTB) 이벤트에 대한 지원을 추가하여 트리거 이벤트 규칙에 대한 기존 간소화된 문법 보기를 개선했습니다. 이전에는 사용자가 사용자 정의 이벤트에 대해 간소화된 문법만 사용할 수 있었고 OOTB와 사용자 정의 이벤트를 개별적으로 구성하려면 &quot;WHEN&quot;과 &quot;ON TRIGGER EVENT&quot; 규칙 간을 전환해야 했습니다. 이 업데이트를 통해 OOTB와 사용자 지정 이벤트를 동일한 간소화된 문법으로 사용할 수 있으므로 규칙 구성을 간소화하고 컨텍스트를 전환할 필요성을 줄일 수 있습니다.
* FORMS-24462: Headless 적응형 Forms(AF)에 대한 React Vanilla 구성 요소에 스크리블 서명 구성 요소에 대한 지원이 추가되었습니다. 이 향상된 기능을 통해 사용자는 React 기반 양식 내에서 직접 친필 서명을 캡처할 수 있으므로 기업 고객을 위한 디지털 서명 워크플로 및 예정된 라이브 타임라인을 지원합니다.
* FORMS-24343: 양식 모델 JavaScript JSON(개체 표기법)에 `custom:setProperty`에 대한 최적화된 처리가 추가되어 동적 속성 업데이트를 더 빨리 처리할 수 있습니다. 이 향상된 기능은 빈번한 런타임 변경에 의존하는 복잡한 적응형 Forms(AF)의 성능을 향상시켜 사용자 상호 작용을 원활하게 하고 로드 시간을 단축합니다.
* FORMS-24358: 모델 JavaScript JSON(Object Notation) 구조에서 `items` 및 `:items` 대신 `:itemsOrder` 속성을 사용할 수 있도록 지원을 추가했습니다. 이 향상된 기능을 통해 개발자는 일반적인 JSON 규칙에 더 잘 부합하고 외부 시스템과의 통합을 간소화하는 보다 깔끔하고 직관적인 데이터 모델을 사용하여 작업할 수 있습니다.
* FORMS-24087: 적응형 Forms(AF)의 조각 컨테이너에서 직접 규칙 및 이벤트를 정의하는 지원이 추가되었습니다. 이 향상된 기능을 통해 작성자는 컨테이너 수준에서 조건부 논리 및 상호 작용을 적용할 수 있으므로 재사용이 향상되고 개별 조각 필드에서 규칙을 복제할 필요가 없습니다.
* FORMS-24440: 대화형 통신 편집기용 규칙 편집기의 THEN 드롭다운에 규칙 조건이 충족될 때 사용자가 양식에서 선택한 구성 요소를 완전히 제거할 수 있는 새로운 &quot;필드 제거&quot; 작업을 추가했습니다. 이 향상된 기능은 일관된 동작을 위해 적절한 `forms_ready` 스크립트를 트리거하면서 필드만 숨기는 대신 양식을 동적으로 재구성해야 하는 워크플로를 지원합니다.
* FORMS-23898: 대화형 통신(IC) 편집기에서 `@` 표기법을 사용하여 변수를 정의하는 기능이 추가되어 사용자가 다이내믹 테이블을 보다 직관적으로 구성할 수 있습니다. 이 향상된 기능을 통해 변수 기반 테이블 콘텐츠를 간단하게 설정하고 작성 환경에서 동적 데이터를 관리할 때의 명확성을 향상시킬 수 있습니다.
* FORMS-23702: SharePoint 데이터에 보다 안전한 인증서 기반 액세스를 가능하게 하는 SharePoint 목록(SPList) 연결에 인증서 기반 인증을 추가했습니다. 이러한 향상된 기능을 통해 기업 고객은 더욱 엄격한 보안 및 규정 준수 요구 사항을 충족하는 동시에 암호 기반 인증에 대한 의존도를 줄일 수 있습니다.
* FORMS-23800: sling 구성에서 reCAPTCHA 비밀 키 재정의에 대한 지원을 추가하여 기업 고객이 자체 보안 및 규정 준수 요구 사항을 맞출 수 있도록 했습니다. 이 향상된 기능을 통해 환경별 비밀 키 관리를 수행할 수 있으므로 관리자는 코드를 변경하지 않고 reCAPTCHA를 안전하게 통합할 수 있습니다.

* FORMS-24520: 사용자에게 적응형 사용자 인터페이스(AUI)를 사용하는 양식에 대한 에이전트 UI(사용자 인터페이스) 인쇄 미리 보기에서 페이지 번호가 누락되었습니다. 에이전트에서 인쇄 미리 보기를 열면 페이지 번호 필드가 비어 있는 것처럼 보이는 경우가 있어 인쇄된 복사본을 검토하거나 공유하는 동안 특정 페이지를 참조하기가 더 어려워졌습니다.

* FORMS-18631: 데스크탑, 반응형 웹 디자인(RWD) 태블릿 및 RWD 모바일 보기의 그리드 레이아웃에서 사용자에게 접근성 문제가 발생했습니다. Windows 11에서 NVDA(NonVisual Desktop Access) 화면 판독기와 함께 Chrome을 사용할 때 그리드에 적절한 역할과 특성이 없으므로 보조 기술이 콘텐츠를 올바르게 해석하고 탐색하는 것이 어렵습니다.

* FORMS-24334: Adobe Experience Manager(AEM) Forms as a Cloud Service에서 포함된 AF(적응형 양식)로 작업할 때 사용자에게 미리 채우기 실패 및 JavaScript 개체 표기법(JSON) 병합 문제가 발생했습니다. 마이그레이션된 양식을 로드할 때 필요한 미리 채워진 데이터가 표시되지 않았고 병합된 JSON 콘텐츠가 불완전하거나 올바르지 않습니다. 이로 인해 영향을 받는 환경에 대해 온-프레미스 AEM 6.5에서 AEM Forms as a Cloud Service으로 마이그레이션이 차단되었습니다.
* FORMS-24441: 사용자에게 Adobe Experience Manager(AEM) Forms as a Cloud Service의 DoR(Document of Record) 템플릿 구성 문제가 발생했습니다. 빠른 개발 환경에서 사용자 지정된 DoR 템플릿을 저장할 때 템플릿이 기본 버전으로 복구되어 의도한 레이아웃과 설정을 유지할 수 없습니다.
* FORMS-24393: 이전 템플릿이 의미 있는 이름을 표시하는 대신 &quot;제목 없음&quot;으로 계속 표시되면서 사용자가 혼란을 경험했습니다. 이로 인해 일상적인 저작 작업 시 기존 템플릿을 구분하고 재사용하기 어려웠다.
* FORMS-24163: 조각이 포함된 버전 2 양식을 미리 볼 때 문제가 발생했습니다. 미리보기 모드에서 양식 콘텐츠가 예상대로 렌더링되지 않아 사용자가 게시하기 전에 레이아웃 및 동작의 유효성을 검사할 수 없습니다.
* FORMS-24328: &quot;사용자 작업에 대한 CAPTCHA 유효성 검사&quot; 옵션과 함께 보이지 않는 reCAPTCHA v2를 사용할 때 양식 제출이 완료되지 않은 경험이 있습니다. 엔터프라이즈 고객은 영향을 받는 환경의 양식이 예상대로 제출되지 않아 연락처 및 제안 요청 워크플로우가 중단되는 것을 확인했습니다.
