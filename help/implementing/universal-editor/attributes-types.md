# 속성 및 유형 {#attributes-types}

범용 편집기에 필요한 데이터 속성 및 유형에 대해 알아봅니다.

## 소개 {#introduction}

범용 편집기에서 앱을 편집할 수 있으려면 제대로 구현되어야 합니다. 여기에는 편집기에서 앱 콘텐츠를 편집할 수 있도록 적절한 메타데이터가 포함됩니다. 이 문서에서는 이러한 메타데이터의 속성 및 유형에 대해 자세히 설명합니다.

>[!NOTE]
>
>서버 측에서 컨텐츠 유효성 검사가 수행됩니다. 범용 편집기는 데이터 속성과 함께 작동합니다. 모델/구조에 맞는지 확인하는 방법은 API 수준에서 지정해야 합니다.

## 데이터 속성 {#data-properties}

| 데이터 속성 | 설명 |
|---|---|
| `itemid` | URN을 리소스에 추가합니다. 섹션을 참조하십시오 [AEM에서 범용 편집기 시작하기 문서의 페이지 악기](getting-started.md#instrument-thepage) |
| `itemprop` | 리소스의 속성입니다. 섹션을 참조하십시오 [AEM에서 범용 편집기 시작하기 문서의 페이지 악기](getting-started.md#instrument-thepage) |
| `itemtype` | 편집 가능한 항목의 유형(예: 텍스트, 이미지, 참조 등) |
| `data-editor-itemfilter` | 사용할 수 있는 참조를 정의합니다 |
| `data-editor-itemlabel` | 편집기에 표시되는 선택 가능한 항목에 대한 사용자 지정 레이블을 정의합니다 <br>경우에 따라 `itemmodel` 이(가) 설정되어 있으면 모델을 통해 레이블이 검색됩니다 |
| `data-editor-itemmodel` | 속성 레일에서 양식 기반 편집에 사용할 모델을 정의합니다 |
| `data-editor-behavior` | 계기의 동작을 정의합니다(예: 독립형 텍스트 또는 이미지도 구성 요소를 모방하여 이동 가능하게 하거나 삭제할 수 있음). |

## 항목 유형 {#item-types}

| `itemtype` | 설명 | `itemid` | `itemprop` | `data-editor-itemfilter` | `data-editor-itemlabel` | `data-editor-itemmodel` | `data-editor-behvior` |
|---|---|---|---|---|---|---|---|
| `text` | 텍스트는 HTML 태그 내에서 편집할 수 있지만 단순 텍스트 형식으로만 사용할 수 있고 리치 텍스트 서식을 사용할 수 없습니다. 일반적으로 제목 구성 요소에 사용됩니다 | 선택 사항 | 필수 | 해당 없음 | 선택 사항 | 해당 없음 | 선택 사항 |
| `richtext` | 텍스트는 전체 리치 텍스트 기능으로 편집할 수 있습니다. RTE가 오른쪽 패널에 표시됩니다 | 선택 사항 | 필수 | 해당 없음 | 선택 사항 | 해당 없음 | 선택 사항 |
| `media` | 편집 가능한 항목은 이미지 또는 비디오와 같은 자산입니다 | 선택 사항 | 필수 | 선택 사항입니다<br>자산 선택기에 전달되는 이미지 또는 비디오 필터 기준 목록 | 선택 사항 | 해당 없음 | 선택 사항 |
| `container` | 편집 가능 항목은 단락 시스템이라는 구성 요소에 대한 컨테이너로 동작합니다. | 상황에 따라 <br>아래 참조 | 상황에 따라 <br>아래 참조 | 선택 사항입니다<br>허용된 구성 요소 목록 | 선택 사항 | 해당 없음 | 해당 없음 |
| `component` | 편집 가능한 항목은 구성 요소입니다. 는 추가 기능을 추가하지 않습니다. 는 DOM의 이동/삭제 가능 부분을 나타내고 속성 레일과 해당 필드를 여는 데 필요합니다 | 필수 | 해당 없음 | 해당 없음 | 선택 사항 | 선택 사항 | 해당 없음 |
| `reference` | 편집 가능한 항목은 참조(예: 컨텐츠 조각, 경험 조각 또는 제품)입니다 | 상황에 따라 <br>아래 참조 | 상황에 따라 <br>아래 참조 | 선택 사항입니다<br>참조 선택기로 전달되는 컨텐츠 조각, 제품 또는 경험 조각 필터 기준 목록 | 선택 사항 | 선택 사항 | 해당 없음 |

사용 사례에 따라 `itemprop` 또는 `itemid` 필수가 되거나 필수가 아닐 수 있습니다. 예:

* `itemid` GraphQL을 통해 컨텐츠 조각을 쿼리하고 목록을 컨텍스트 내에서 편집할 수 있도록 하려는 경우 필요합니다.
* `itemprop` 는 참조된 컨텐츠 조각의 컨텐츠를 렌더링하는 구성 요소가 있고 구성 요소 내에서 참조를 업데이트하려는 경우 필요합니다.

## 비헤이비어 {#behaviors}

| `data-editor-behavior` | 설명 |
|---|---|
| `component` | 독립 실행형 텍스트, 리치 텍스트 및 미디어를 통해 구성 요소를 모방하도록 하여 페이지에서 이동 및 삭제할 수도 있습니다 |
